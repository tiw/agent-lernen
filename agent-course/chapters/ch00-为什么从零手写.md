# 第 0 章：为什么我们要从零手写？

> **本章目标**：理解为什么不用框架、了解课程全貌、搭建开发环境

---

## 🔍 先看 Claude Code 怎么做

Claude Code 是 Anthropic 出品的 AI 编码助手，它的源码（我们在本地有 `claude-code-main`）有 **1332 个源文件**，但核心架构极其清晰：

```
claude-code-main/src/
├── tools/        # 40+ 个工具（文件、Shell、搜索、MCP...）
├── tasks/        # 任务系统（本地/远程 Agent、Shell 任务）
├── services/     # 核心服务（记忆、Token 估算、MCP...）
├── context/      # 上下文管理
├── skills/       # 技能系统
├── hooks/        # 钩子系统
├── coordinator/  # 多智能体协调
└── cli/          # 终端界面
```

**关键发现**：Claude Code 没有使用 LangChain、LlamaIndex 或任何 Agent 框架。它是从零构建的。为什么？

### Claude Code 不用框架的三个原因

1. **框架抽象泄漏**：LangChain 的 Chain/Agent 抽象在简单场景好用，但一旦需要精细控制（比如 Claude Code 的 40+ 工具调度），框架反而成了障碍
2. **调试噩梦**：框架层叠的抽象让调试变得困难。Claude Code 需要精确控制每个工具的输入输出、错误处理、重试策略
3. **性能瓶颈**：框架的通用性带来性能开销。Claude Code 需要毫秒级的工具调用响应

## 🧠 核心概念

### 什么是 Agent？

Agent 的本质公式：

```
Agent = LLM + 循环 + 工具 + 记忆
```

- **LLM**：大脑，负责思考和决策
- **循环**：持续运行，直到任务完成
- **工具**：手和脚，让 AI 能执行实际操作
- **记忆**：经验积累，跨会话保持上下文

### 框架 vs 手写的对比

| 维度 | 框架（LangChain 等） | 从零手写 |
|---|---|---|
| 上手速度 | 快（几行代码跑起来） | 慢（需要理解每个组件） |
| 控制力 | 低（框架决定你能做什么） | 高（你决定一切） |
| 调试难度 | 高（框架内部黑盒） | 低（每行代码都是你的） |
| 性能 | 中（框架开销） | 高（无多余抽象） |
| 可扩展性 | 受限 | 无限 |
| 面试/竞争力 | 一般 | 强 |

### 本课程的学习哲学

```
理解原理 → 手写实现 → 跑通验证 → 迭代优化
```

每一章都遵循这个循环。我们不追求"先跑起来再说"，而是追求"真正理解每一行代码"。

## 💻 环境搭建

### 1. Python 环境

```bash
# 确认 Python 版本（需要 3.10+）
python3 --version

# 创建项目目录
mkdir my-agent && cd my-agent

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate  # macOS/Linux
# 或 .venv\Scripts\activate  # Windows
```

### 2. 安装依赖

```bash
pip install openai anthropic httpx aiosqlite rich click
```

| 包 | 用途 |
|---|---|
| `openai` | OpenAI API 客户端 |
| `anthropic` | Anthropic API 客户端 |
| `httpx` | 异步 HTTP 客户端 |
| `aiosqlite` | 异步 SQLite |
| `rich` | 终端美化输出 |
| `click` | CLI 命令行框架 |

### 3. 配置 API 密钥

```bash
# OpenAI
export OPENAI_API_KEY="sk-..."

# 或 Anthropic
export ANTHROPIC_API_KEY="sk-ant-..."
```

### 4. 项目结构

```
my-agent/
├── agent.py          # Agent 核心
├── tools/            # 工具系统
│   ├── __init__.py
│   ├── base.py
│   └── ...
├── memory/           # 记忆系统
├── tasks/            # 任务系统
├── skills/           # 技能系统
├── mcp/              # MCP 协议
├── team/             # 多智能体
├── hooks/            # 钩子系统
├── security/         # 安全系统
├── cli/              # 终端界面
└── tests/            # 测试
```

## 📝 本章小结

- Claude Code 从零构建，不使用任何 Agent 框架
- 框架适合快速原型，但不适合深入理解和生产级系统
- Agent = LLM + 循环 + 工具 + 记忆
- 本课程将带你从零实现一个完整的 Agent 系统
