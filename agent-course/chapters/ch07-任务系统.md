# 第 7 章：任务系统 —— 让 AI 同时做多件事

> 一个优秀的 Agent 不应该傻傻地等一个命令执行完才做下一件事。真正的智能体，能同时管理多个后台任务、委派子 Agent、随时暂停和恢复。这就是**任务系统**的价值。

---

## 🔍 先看 Claude Code 怎么做

Claude Code 的任务系统是整个架构的核心骨架之一。打开源码 `src/Task.ts`，你会看到一个精心设计的基础层：

```typescript
// src/Task.ts —— 任务类型与状态定义
export type TaskType =
  | 'local_bash'        // 本地 Shell 任务
  | 'local_agent'       // 本地子 Agent 任务
  | 'remote_agent'      // 远程 Agent 任务
  | 'in_process_teammate' // 进程内队友
  | 'local_workflow'    // 本地工作流
  | 'monitor_mcp'       // MCP 监控任务
  | 'dream'             // 梦境任务（后台思考）

export type TaskStatus =
  | 'pending'           // 等待中
  | 'running'           // 运行中
  | 'completed'         // 已完成
  | 'failed'            // 已失败
  | 'killed'            // 已终止
```

所有任务共享一个基类 `TaskStateBase`：

```typescript
export type TaskStateBase = {
  id: string              // 唯一 ID（带类型前缀，如 b1a2c3d4、a5e6f7g8）
  type: TaskType          // 任务类型
  status: TaskStatus      // 当前状态
  description: string     // 人类可读的描述
  toolUseId?: string      // 关联的工具调用 ID
  startTime: number       // 创建时间戳
  endTime?: number        // 结束时间戳
  outputFile: string      // 输出文件路径
  outputOffset: number    // 输出读取偏移量
  notified: boolean       // 是否已发送通知
}
```

任务接口极其简洁——只有 `kill` 方法：

```typescript
export type Task = {
  name: string
  type: TaskType
  kill(taskId: string, setAppState: SetAppState): Promise<void>
}
```

> **为什么只有 kill？** 因为 spawn（创建）和 render（渲染）从未被多态调用。每种任务类型有自己的 spawn 函数，但 kill 是统一的——所有任务都能被终止。

### Shell 任务：LocalShellTask

`src/tasks/LocalShellTask/LocalShellTask.tsx` 实现了后台 Shell 执行，关键设计：

1. **前台/后台切换**：任务可以先在前台运行，运行时间过长后自动切到后台（`registerForeground` → `backgroundTask`）
2. **卡死检测（Stall Watchdog）**：每 5 秒检查一次输出文件，如果 45 秒没有增长且尾部看起来像交互提示符（如 `(y/n)`、`Continue?`），就通知用户
3. **通知系统**：任务完成/失败/被杀时，通过 XML 标签发送结构化通知

### Agent 任务：LocalAgentTask

`src/tasks/LocalAgentTask/LocalAgentTask.tsx` 实现了子 Agent 委派：

1. **AbortController 层级**：子 Agent 的 AbortController 可以是父控制器的子控制器，父取消时子自动取消
2. **进度追踪**：记录 toolUseCount、tokenCount、最近活动
3. **消息队列**：支持 `pendingMessages`，在工具轮次边界排空
4. **前后台切换**：和 Shell 任务一样支持 foreground → background

### 任务停止：stopTask.ts

`src/tasks/stopTask.ts` 提供了统一的停止逻辑：

```typescript
export async function stopTask(taskId: string, context: StopTaskContext) {
  // 1. 查找任务
  // 2. 验证状态（必须 running）
  // 3. 获取 Task 实现
  // 4. 调用 taskImpl.kill()
  // 5. 标记 notified（抑制重复通知）
}
```

## 🧠 核心概念

### 任务系统架构

```
┌─────────────────────────────────────────────┐
│              Task Registry                   │
│   tasks: { taskId → TaskState }             │
├──────────┬──────────┬───────────────────────┤
│ ShellTask│ AgentTask│   (可扩展更多类型)     │
│          │          │                       │
│ 异步执行  │ 子Agent  │                       │
│ 命令     │ 委派执行  │                       │
├──────────┴──────────┴───────────────────────┤
│            TaskStateBase                     │
│  id | type | status | description | ...     │
├─────────────────────────────────────────────┤
│            状态机                            │
│  pending → running → completed/failed/killed│
└─────────────────────────────────────────────┘
```

### 状态机

```
                    ┌─────────┐
         create     │ pending  │
         ─────────► │         │
                    └────┬────┘
                         │ start
                         ▼
                    ┌─────────┐
               ┌─── │ running │ ───┐
               │    │         │    │
               │    └────┬────┘    │
               │         │         │
            pause    自然结束    kill
               │         │         │
               ▼         ▼         ▼
         ┌─────────┐ ┌─────────┐ ┌─────────┐
         │ pending │ │completed│ │ killed  │
         │(可恢复)  │ │(终态)   │ │(终态)   │
         └─────────┘ └─────────┘ └─────────┘
                         │
                         ▼
                    ┌─────────┐
                    │ failed  │
                    │(终态)   │
                    └─────────┘
```

### 关键设计原则

1. **类型前缀 ID**：不同任务类型有不同 ID 前缀（`b`=bash, `a`=agent），便于快速识别
2. **输出文件驱动**：任务输出写入磁盘文件，UI 通过 tail 文件读取，解耦执行和展示
3. **通知去重**：`notified` 标志防止重复通知
4. **清理注册表**：每个任务注册清理回调，确保资源不泄漏

### 1. 任务基类 `tasks/base.py`

```python
"""
tasks/base.py —— 任务系统核心：基类、状态机、注册表

参考 Claude Code 的 Task.ts / TaskStateBase / stopTask.ts
"""

import uuid
import time
import asyncio
import logging
from abc import ABC, abstractmethod
from enum import Enum
from dataclasses import dataclass, field
from typing import Optional, Callable, Any

logger = logging.getLogger(__name__)


class TaskType(Enum):
    """任务类型（参考 Claude Code 的 TaskType）"""
    SHELL = "shell"
    AGENT = "agent"


class TaskStatus(Enum):
    """任务状态机（参考 Claude Code 的 TaskStatus）"""
    PENDING = "pending"       # 等待中
    RUNNING = "running"       # 运行中
    COMPLETED = "completed"   # 已完成
    FAILED = "failed"         # 已失败
    KILLED = "killed"         # 已终止（用户主动停止）

    @property
    def is_terminal(self) -> bool:
        """是否处于终态（不再转换）"""
        return self in (TaskStatus.COMPLETED, TaskStatus.FAILED, TaskStatus.KILLED)


# 任务 ID 前缀（参考 Claude Code 的 TASK_ID_PREFIXES）
TASK_ID_PREFIXES = {
    TaskType.SHELL: "b",
    TaskType.AGENT: "a",
}


def generate_task_id(task_type: TaskType) -> str:
    """
    生成带类型前缀的任务 ID。
    参考 Claude Code: prefix + 8 个随机字符
    """
    prefix = TASK_ID_PREFIXES.get(task_type, "x")
    short_uuid = uuid.uuid4().hex[:8]
    return f"{prefix}{short_uuid}"


@dataclass
class TaskState:
    """
    任务状态（参考 Claude Code 的 TaskStateBase）

    所有任务类型共享的字段。
    """
    id: str
    type: TaskType
    status: TaskStatus = TaskStatus.PENDING
    description: str = ""
    start_time: float = 0.0
    end_time: Optional[float] = None
    output: str = ""
    error: Optional[str] = None
    notified: bool = False  # 是否已发送完成通知

    def __post_init__(self):
        if self.start_time == 0.0:
            self.start_time = time.time()


class Task(ABC):
    """
    任务基类（参考 Claude Code 的 Task 接口）

    核心设计：
    - 每种任务类型实现自己的 spawn（创建）逻辑
    - kill 是统一的接口——所有任务都能被终止
    - 状态转换通过 TaskRegistry 统一管理
    """

    def __init__(self, task_id: str, description: str):
        self.task_id = task_id
        self.description = description
        self.state = TaskState(
            id=task_id,
            type=self.task_type,
            description=description,
        )
        self._cleanup_callbacks: list[Callable] = []

    @property
    @abstractmethod
    def task_type(self) -> TaskType:
        """返回任务类型"""
        ...

    @abstractmethod
    async def run(self) -> str:
        """
        执行任务，返回输出字符串。
        子类实现具体逻辑。
        """
        ...

    async def kill(self) -> None:
        """
        终止任务（参考 Claude Code 的 Task.kill）。
        默认实现：设置状态为 killed。
        子类可以覆盖以执行额外清理。
        """
        if not self.state.status.is_terminal:
            self.state.status = TaskStatus.KILLED
            self.state.end_time = time.time()
            logger.info(f"Task {self.task_id} killed")
        await self._run_cleanup()

    def register_cleanup(self, callback: Callable) -> None:
        """注册清理回调（参考 Claude Code 的 registerCleanup）"""
        self._cleanup_callbacks.append(callback)

    async def _run_cleanup(self) -> None:
        """执行所有清理回调"""
        for cb in self._cleanup_callbacks:
            try:
                if asyncio.iscoroutinefunction(cb):
                    await cb()
                else:
                    cb()
            except Exception as e:
                logger.warning(f"Cleanup callback error: {e}")
        self._cleanup_callbacks.clear()


class TaskRegistry:
    """
    任务注册表（参考 Claude Code 的 tasks 状态管理）

    管理所有任务的生命周期，提供统一的创建、查询、停止接口。
    """

    def __init__(self):
        self._tasks: dict[str, Task] = {}
        self._running_tasks: set[str] = set()
        self._notification_callbacks: list[Callable] = []

    def on_notification(self, callback: Callable[[dict], None]) -> None:
        """注册通知回调"""
        self._notification_callbacks.append(callback)

    def _notify(self, task_id: str, event: str, data: dict) -> None:
        """发送任务通知（去重）"""
        task = self._tasks.get(task_id)
        if task and task.state.notified:
            return  # 已通知，跳过
        if task:
            task.state.notified = True
        msg = {"task_id": task_id, "event": event, **data}
        for cb in self._notification_callbacks:
            try:
                cb(msg)
            except Exception as e:
                logger.warning(f"Notification callback error: {e}")

    def register(self, task: Task) -> None:
        """注册任务到注册表"""
        self._tasks[task.task_id] = task
        task.state.status = TaskStatus.PENDING
        logger.info(f"Task registered: {task.task_id} ({task.task_type.value})")

    async def start(self, task_id: str) -> asyncio.Task:
        """
        启动任务（参考 Claude Code 的 spawnShellTask / registerAsyncAgent）

        返回 asyncio.Task 供外部 await 或后台运行。
        """
        task = self._tasks.get(task_id)
        if not task:
            raise ValueError(f"Task not found: {task_id}")
        if task.state.status != TaskStatus.PENDING:
            raise ValueError(f"Task {task_id} is not pending (status: {task.state.status.value})")

        task.state.status = TaskStatus.RUNNING
        self._running_tasks.add(task_id)

        async def _run_wrapper():
            try:
                output = await task.run()
                if task.state.status == TaskStatus.RUNNING:
                    task.state.status = TaskStatus.COMPLETED
                    task.state.output = output
                    task.state.end_time = time.time()
                    self._notify(task_id, "completed", {
                        "output": output[:500],  # 截断避免过大
                        "duration": task.state.end_time - task.state.start_time,
                    })
            except asyncio.CancelledError:
                task.state.status = TaskStatus.KILLED
                task.state.end_time = time.time()
                self._notify(task_id, "killed", {})
            except Exception as e:
                task.state.status = TaskStatus.FAILED
                task.state.error = str(e)
                task.state.end_time = time.time()
                self._notify(task_id, "failed", {"error": str(e)})
            finally:
                self._running_tasks.discard(task_id)
                await task._run_cleanup()

        return asyncio.create_task(_run_wrapper(), name=f"task-{task_id}")

    async def stop(self, task_id: str) -> dict:
        """
        停止任务（参考 Claude Code 的 stopTask）

        统一入口：查找 → 验证 → 终止 → 通知
        """
        task = self._tasks.get(task_id)
        if not task:
            raise ValueError(f"No task found with ID: {task_id}")
        if task.state.status == TaskStatus.PENDING:
            # pending 任务直接标记为 killed
            task.state.status = TaskStatus.KILLED
            task.state.end_time = time.time()
            self._notify(task_id, "killed", {})
            return {"task_id": task_id, "status": "killed"}
        if task.state.status != TaskStatus.RUNNING:
            raise ValueError(f"Task {task_id} is not running (status: {task.state.status.value})")

        await task.kill()
        self._running_tasks.discard(task_id)
        return {"task_id": task_id, "status": "killed"}

    def get(self, task_id: str) -> Optional[Task]:
        return self._tasks.get(task_id)

    def list_tasks(self) -> list[dict]:
        """列出所有任务状态"""
        return [
            {
                "id": t.task_id,
                "type": t.task_type.value,
                "status": t.state.status.value,
                "description": t.description,
            }
            for t in self._tasks.values()
        ]

    @property
    def running_count(self) -> int:
        return len(self._running_tasks)
```

### 2. Shell 任务 `tasks/shell_task.py`

```python
"""
tasks/shell_task.py —— 异步 Shell 任务

参考 Claude Code 的 LocalShellTask：
- 后台执行 Shell 命令
- 支持超时
- 输出实时收集
- 卡死检测（简化版）
"""

import asyncio
import logging
import time
from typing import Optional

from .base import Task, TaskType, generate_task_id

logger = logging.getLogger(__name__)


class ShellTask(Task):
    """
    异步 Shell 任务

    参考 Claude Code 的 LocalShellTask：
    - 命令在后台异步执行
    - 输出实时收集到 state.output
    - 支持超时自动终止
    - 支持前台/后台模式
    """

    def __init__(
        self,
        command: str,
        description: str = "",
        timeout: Optional[float] = None,
        task_id: Optional[str] = None,
        background: bool = True,
    ):
        tid = task_id or generate_task_id(TaskType.SHELL)
        super().__init__(tid, description or command)
        self.command = command
        self.timeout = timeout
        self.background = background
        self._process: Optional[asyncio.subprocess.Process] = None
        self._killed = False

    @property
    def task_type(self) -> TaskType:
        return TaskType.SHELL

    async def run(self) -> str:
        """
        异步执行 Shell 命令

        参考 Claude Code 的 spawnShellTask：
        1. 创建子进程
        2. 实时读取输出
        3. 处理超时
        4. 返回完整输出
        """
        logger.info(f"ShellTask {self.task_id}: running '{self.command}'")

        self._process = await asyncio.create_subprocess_shell(
            self.command,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.STDOUT,
        )

        # 注册清理：进程异常退出时清理
        self.register_cleanup(self._cleanup_process)

        try:
            # 实时读取输出
            output_lines = []
            async for line in self._process.stdout:
                decoded = line.decode("utf-8", errors="replace")
                output_lines.append(decoded)
                self.state.output = "".join(output_lines)

                # 如果已被 kill，停止读取
                if self._killed:
                    break

            # 等待进程结束
            if not self._killed:
                await self._process.wait()

            return self.state.output

        except asyncio.TimeoutError:
            if self._process:
                self._process.kill()
                await self._process.wait()
            raise TimeoutError(
                f"Shell command timed out after {self.timeout}s: {self.command}"
            )

    async def kill(self) -> None:
        """终止 Shell 进程（参考 Claude Code 的 killTask）"""
        self._killed = True
        if self._process and self._process.returncode is None:
            try:
                self._process.kill()
                await self._process.wait()
            except ProcessLookupError:
                pass  # 进程已结束
        await super().kill()

    async def _cleanup_process(self) -> None:
        """清理子进程"""
        if self._process and self._process.returncode is None:
            try:
                self._process.kill()
            except ProcessLookupError:
                pass
```

### 3. Agent 任务 `tasks/agent_task.py`

```python
"""
tasks/agent_task.py —— 子 Agent 委派任务

参考 Claude Code 的 LocalAgentTask：
- 创建子 Agent 执行复杂任务
- 支持 AbortController 层级（父取消 → 子自动取消）
- 进度追踪（tool_use_count, token_count）
- 消息队列
"""

import asyncio
import logging
import time
from dataclasses import dataclass, field
from typing import Optional, Callable, Any

from .base import Task, TaskType, generate_task_id

logger = logging.getLogger(__name__)


@dataclass
class AgentProgress:
    """
    Agent 进度信息（参考 Claude Code 的 AgentProgress）
    """
    tool_use_count: int = 0
    token_count: int = 0
    last_activity: str = ""
    summary: str = ""


@dataclass
class AgentDefinition:
    """
    Agent 定义（参考 Claude Code 的 AgentDefinition）
    """
    name: str
    system_prompt: str
    model: str = "default"
    max_turns: int = 10
    allowed_tools: list[str] = field(default_factory=list)


class AgentTask(Task):
    """
    子 Agent 委派任务

    参考 Claude Code 的 LocalAgentTask：
    - 子 Agent 独立运行
    - 支持父 Agent 取消时自动取消子 Agent
    - 实时进度更新
    - 消息队列支持
    """

    def __init__(
        self,
        agent_def: AgentDefinition,
        prompt: str,
        description: str = "",
        task_id: Optional[str] = None,
        parent_cancel_token: Optional[asyncio.Event] = None,
    ):
        tid = task_id or generate_task_id(TaskType.AGENT)
        super().__init__(tid, description or f"Agent: {agent_def.name}")
        self.agent_def = agent_def
        self.prompt = prompt
        self.parent_cancel_token = parent_cancel_token
        self.progress = AgentProgress()
        self.pending_messages: list[str] = []
        self._cancel_event = asyncio.Event()
        self._current_turn = 0

    @property
    def task_type(self) -> TaskType:
        return TaskType.AGENT

    async def run(self) -> str:
        """
        执行子 Agent

        参考 Claude Code 的 registerAsyncAgent + AgentTool：
        1. 检查父取消信号
        2. 执行 Agent 循环
        3. 更新进度
        4. 返回最终结果
        """
        logger.info(f"AgentTask {self.task_id}: starting '{self.agent_def.name}'")

        # 监听父取消信号
        if self.parent_cancel_token:
            asyncio.create_task(self._watch_parent_cancel())

        try:
            result = await self._agent_loop()
            return result
        except asyncio.CancelledError:
            logger.info(f"AgentTask {self.task_id}: cancelled")
            raise

    async def _agent_loop(self) -> str:
        """
        Agent 主循环（简化版）

        真实实现会调用 LLM API、解析工具调用、执行工具。
        这里用模拟展示架构。
        """
        messages = [
            {"role": "system", "content": self.agent_def.system_prompt},
            {"role": "user", "content": self.prompt},
        ]

        result_parts = []

        for turn in range(1, self.agent_def.max_turns + 1):
            self._current_turn = turn

            # 检查取消
            if self._cancel_event.is_set():
                raise asyncio.CancelledError("Agent cancelled")

            # 处理待处理消息
            while self.pending_messages:
                msg = self.pending_messages.pop(0)
                messages.append({"role": "user", "content": msg})

            # 模拟 LLM 调用
            response = await self._call_llm(messages)
            result_parts.append(f"[Turn {turn}] {response}")

            # 更新进度
            self.progress.tool_use_count += 1
            self.progress.token_count += len(response)
            self.progress.last_activity = f"Turn {turn} completed"

            # 如果响应包含完成标记，结束循环
            if "[DONE]" in response:
                break

            messages.append({"role": "assistant", "content": response})

        return "\n".join(result_parts)

    async def _call_llm(self, messages: list[dict]) -> str:
        """
        调用 LLM（模拟实现）

        真实实现应该调用 OpenAI/Anthropic API。
        """
        last_msg = messages[-1]["content"]
        # 模拟：根据内容生成不同响应
        if "search" in last_msg.lower() or "find" in last_msg.lower():
            await asyncio.sleep(0.5)  # 模拟网络延迟
            return f"Searching for information about: {last_msg[:50]}... [DONE]"
        elif "write" in last_msg.lower() or "create" in last_msg.lower():
            await asyncio.sleep(0.3)
            return f"Creating content based on: {last_msg[:50]}... [DONE]"
        else:
            await asyncio.sleep(0.2)
            return f"Processed: {last_msg[:50]}... [DONE]"

    async def _watch_parent_cancel(self) -> None:
        """
        监听父 Agent 取消信号（参考 Claude Code 的 createChildAbortController）

        当父 Agent 被取消时，子 Agent 自动取消。
        """
        if self.parent_cancel_token:
            await self.parent_cancel_token.wait()
            logger.info(f"AgentTask {self.task_id}: parent cancelled, stopping")
            self._cancel_event.set()

    def send_message(self, message: str) -> None:
        """
        向运行中的 Agent 发送消息（参考 Claude Code 的 queuePendingMessage）
        """
        self.pending_messages.append(message)
        logger.info(f"AgentTask {self.task_id}: message queued")

    async def kill(self) -> None:
        """终止 Agent 任务"""
        self._cancel_event.set()
        await super().kill()
```

## 📝 本章小结

本章我们深入理解了 Claude Code 的任务系统架构，并用 Python 从零实现了一个完整的任务管理系统：

| 概念 | Claude Code 实现 | 我们的 Python 实现 |
|------|-----------------|-------------------|
| 任务类型 | TaskType 枚举（7 种） | TaskType 枚举（2 种起步） |
| 状态机 | pending→running→终态 | 相同的状态机 |
| 任务基类 | Task 接口（name, type, kill） | Task 抽象基类 |
| Shell 任务 | LocalShellTask（异步+卡死检测） | ShellTask（异步+超时） |
| Agent 任务 | LocalAgentTask（AbortController+进度） | AgentTask（Event+进度） |
| 注册表 | AppState.tasks 字典 | TaskRegistry 类 |
| 停止逻辑 | stopTask.ts（统一入口） | registry.stop() |

**核心收获：**

1. **任务系统的本质是状态管理**——所有复杂性都围绕"如何让状态正确转换"展开
2. **异步是关键**——没有 asyncio，任务系统就无法实现真正的并发
3. **清理必须可靠**——registerCleanup 模式确保资源不泄漏
4. **通知去重很重要**——notified 标志防止重复通知

### 练习 1：实现任务暂停/恢复

> **提示**：在 TaskStatus 中添加 `PAUSED` 状态，在 TaskRegistry 中实现 `pause()` 和 `resume()` 方法。Shell 任务可以用 `SIGSTOP`/`SIGCONT` 信号实现。

### 练习 2：实现卡死检测（Stall Watchdog）

> **提示**：参考 Claude Code 的 `startStallWatchdog`，启动一个定时器，每 5 秒检查输出是否增长。如果 45 秒没有增长，发送卡死通知。可以用 `re` 模块检测输出尾部是否像交互提示符。

### 练习 3：实现任务优先级队列

> **提示**：在 TaskState 中添加 `priority: int` 字段，修改 TaskRegistry 的 `start` 方法，优先启动高优先级任务。可以用 `heapq` 实现优先级队列。

### 练习 4：实现任务依赖

> **提示**：在 TaskState 中添加 `dependencies: list[str]`，只有当所有依赖任务完成后才启动当前任务。可以用拓扑排序确保无死锁。
