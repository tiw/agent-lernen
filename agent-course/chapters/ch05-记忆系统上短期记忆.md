# 第 5 章：记忆系统（上）—— 短期记忆

> **本章目标**：理解 Claude Code 的七层记忆架构，从零实现工具结果预算、微压缩和会话记忆系统。

---

## 🔍 先看 Claude Code 怎么做

Claude Code 的记忆系统是其最复杂也最精妙的部分。它不是简单的"上下文满了就压缩"，而是构建了一个 **七层防御体系**，每一层都有明确的职责和触发条件。

### 七层记忆架构全景图

在深入每一层之前，先建立整体认知：

```
┌─────────────────────────────────────────────────────────────┐
│  第 1 层：工具结果存储（日常清洁工）                           │
│  预算系统、磁盘持久化、预览截断                                │
├─────────────────────────────────────────────────────────────┤
│  第 2 层：微压缩（日常保洁）                                   │
│  时间清理、缓存编辑压缩、API 级上下文管理                       │
├─────────────────────────────────────────────────────────────┤
│  第 3 层：会话记忆（实时结构化笔记）                            │
│  零 API 压缩、GrowthBook 远程配置                              │
├─────────────────────────────────────────────────────────────┤
│  第 4 层：全压缩（紧急刹车）                                   │
│  9 段摘要、草稿剥离、压缩后修复                                │
├─────────────────────────────────────────────────────────────┤
│  第 5 层：自动记忆提取                                        │
│  4 种记忆类型、MEMORY.md 索引                                 │
├─────────────────────────────────────────────────────────────┤
│  第 6 层：做梦机制                                            │
│  跨会话巩固、锁文件、4 阶段                                   │
├─────────────────────────────────────────────────────────────┤
│  第 7 层：跨代理沟通                                          │
│  分支代理、SendMessage、Agent Summary                         │
└─────────────────────────────────────────────────────────────┘
```

**设计哲学**：前 4 层是短期记忆（本章），后 3 层是长期记忆（下一章）。每一层都是上一层的"安全网"——如果第 1 层没拦住 token 增长，第 2 层接手；如果第 2 层也没拦住，第 3 层接手……直到第 4 层紧急刹车。

本章聚焦 **前 3 层**：工具结果存储、微压缩、会话记忆。

### 第 1 层：工具结果存储

源码路径：`src/tools/` 下的工具结果预算系统

Claude Code 的对话中，**工具结果是最主要的 token 消耗源**。读一个 500 行的文件，一次 Bash 命令的输出，一次 Web 搜索的结果——每个都可能吃掉几千 token。

#### COMPACTABLE_TOOLS 集合

Claude Code 定义了一组"可压缩工具"：

```typescript
// src/tools/ 下的工具预算配置
const COMPACTABLE_TOOLS = new Set([
  'FileRead',    // 文件读取
  'Bash',        // 命令执行
  'Grep',        // 文本搜索
  'Glob',        // 文件匹配
  'WebSearch',   // 网络搜索
  'WebFetch',    // 网页抓取
  'FileEdit',    // 文件编辑
  'FileWrite',   // 文件写入
]);
```

这些工具的结果通常很大，需要特殊处理。

#### 预算策略：预览 + 磁盘持久化

当工具结果超过阈值时，Claude Code 不会把完整结果放进上下文，而是：

1. **完整结果写磁盘**：保存到 `tool-results/<session>/<id>.txt`
2. **上下文只放预览**：前 ~2KB 的截断版本
3. **需要时按需读取**：后续对话中如果 Agent 需要完整内容，再从磁盘读取

```
工具执行 → 结果超过阈值？
  ├─ 是 → 写磁盘 + 返回 2KB 预览
  └─ 否 → 直接返回完整结果
```

#### 状态冻结（pinCacheEdits）

这是 Claude Code 最精妙的设计之一。

**问题**：如果第一轮 API 调用时用了完整预览，第二轮时预览被压缩了——Prompt 的前缀字节就变了，**API 缓存全部失效**。

**解决方案**：一旦决定用预览，就"冻结"这个预览版本。后续所有 API 调用都用同样的预览文本，确保 Prompt 前缀字节一致。

```typescript
// 伪代码：状态冻结
function pinCacheEdits(toolCallId: string, preview: string): void {
  // 将这个预览版本标记为"已冻结"
  // 后续 API 调用必须使用相同的预览
  pinnedPreviews.set(toolCallId, preview);
}
```

这就像写文章时锁定第一段——后续无论怎么修改后面的内容，第一段不变，读者（API 缓存）就能快速定位。

#### 远程调节

每个工具的阈值不是硬编码的，而是通过 **GrowthBook 功能标志** 远程调节：

```typescript
// 通过远程配置调节阈值
const fileReadThreshold = growthbook.getFeatureValue(
  'tool-budget-file-read-threshold',
  25_000  // 默认值
);
```

这意味着 Claude 团队可以在不发布新版本的情况下，调整所有用户的工具预算策略。

### 第 2 层：微压缩

源码路径：`src/services/compact/microCompact.ts`

如果第 1 层是"不让大结果进上下文"，那第 2 层就是"定期清理已经进上下文的旧结果"。

#### 三种机制

**机制 1：基于时间的清理**

距离上次助手消息超过阈值（默认 60 分钟），清理旧的工具结果。

```typescript
// 伪代码：时间阈值清理
const TIME_THRESHOLD_MS = 60 * 60 * 1000; // 60 分钟

if (now - lastAssistantMessage.timestamp > TIME_THRESHOLD_MS) {
  // 清理旧工具结果
  messages = replaceOldToolResults(messages, '[tool result omitted]');
}
```

**机制 2：缓存编辑压缩（cache_edits）**

这是 Claude Code 的核心创新。

**问题**：如果直接在本地消息列表中删除旧工具结果，下次 API 调用时 Prompt 前缀就变了，缓存失效。

**解决方案**：在 **服务器端** 删除旧工具结果，本地消息列表保持不变。API 请求时通过 `context_management` 参数告诉服务器"这些消息的内容要跳过"。

```
本地消息：[msg1, msg2, msg3, msg4, msg5]  ← 不变
API 请求：发送完整消息列表 + 标记 [msg1, msg2] 可跳过
服务器：收到请求，跳过 msg1 和 msg2 的内容
```

这样本地 Prompt 前缀不变，缓存命中率最大化。

**机制 3：API 级上下文管理**

使用 Anthropic API 的 `context_management` 参数，在 API 层面控制哪些消息内容需要处理。

#### 关键设计：只运行主线

Claude Code 有一个重要约束：**微压缩只在主线运行**。

如果分支的子代理修改了全局状态，就会破坏主线程的缓存编辑。这意味着：
- 子代理有独立的上下文窗口
- 子代理的结果通过结构化消息传回主线
- 主线不依赖子代理的内部状态做缓存决策

### 第 3 层：会话记忆

当第 1、2 层都不够用，对话继续增长时，Claude Code 启动第 3 层：**会话记忆**。

#### 触发条件

Claude Code 的会话记忆不是"满了才触发"，而是 **持续监控、渐进式更新**：

```typescript
// 触发条件（源码参数）
const config = {
  minimumMessageTokensToInit: 5000,       // 消息总 token 达到 5000 才初始化
  minimumTokensBetweenUpdate: 3000,       // 两次更新之间至少增长 3000 tokens
  toolCallsBetweenUpdates: 10,            // 或者工具调用次数达到 10 次
};

// 触发逻辑
if (totalTokens >= minimumMessageTokensToInit &&
    (tokensSinceLastUpdate >= minimumTokensBetweenUpdate ||
     toolCallsSinceLastUpdate >= toolCallsBetweenUpdates)) {
  updateSessionMemory();
}
```

#### 零 API 压缩策略

Claude Code 的会话记忆有一个关键设计：**需要压缩时，直接注入现成的摘要，不需要额外调用 LLM**。

```
常规流程：
  对话增长 → 调用 LLM 生成摘要 → 注入摘要  （1 次额外 API 调用）

零 API 策略：
  对话增长 → 注入已有的会话记忆摘要         （0 次额外 API 调用）
```

会话记忆是 **实时结构化笔记**——在对话过程中持续更新，压缩时直接拿来用。

#### Prompt Cache 保存策略

Claude Code 维护 **14 个缓存断点**，用粘性锁存器（sticky latch）管理：

```
断点 0: 系统提示词结束
断点 1: 工具定义结束
断点 2: 技能注入结束
断点 3: 会话记忆摘要
断点 4: 压缩边界标记
断点 5: 保留消息开始
...
断点 13: 最新消息结束
```

**粘性锁存器** 的作用是防止模式切换导致缓存失效。比如从"文件编辑模式"切换到"代码执行模式"时，工具定义部分可能变化。锁存器确保即使工具列表变了，缓存断点之前的内容仍然命中。

**每个设计决策都考虑缓存命中**——这不是优化，是架构原则。

### Token 估算服务

源码路径：`src/services/tokenEstimation.ts`

Claude Code 使用 **三层 Token 计数策略**：

1. **API 精确计数**（首选）：调用 Anthropic API 的 `countTokens` 端点
2. **Haiku 回退计数**：通过小模型 Haiku 进行 token 计数（当 API 不可用时）
3. **粗糙估算**（兜底）：基于字符数的简单估算

```typescript
// 粗糙估算：默认 4 bytes/token
export function roughTokenCountEstimation(
  content: string,
  bytesPerToken: number = 4,
): number {
  return Math.round(content.length / bytesPerToken)
}

// 不同文件类型的 bytes/token 比率不同
export function bytesPerTokenForFileType(fileExtension: string): number {
  switch (fileExtension) {
    case 'json':
    case 'jsonl':
    case 'jsonc':
      return 2  // JSON 中有很多单字符 token（{ } : , "）
    default:
      return 4  // 普通文本
  }
}
```

关键洞察：**JSON 的 token 密度是普通文本的 2 倍**。这是因为 JSON 中大量使用 `{`、`}`、`:`、`,`、`"` 等标点符号，每个都是独立的 token。

## 🧠 核心概念

### 1. 七层架构的设计哲学

Claude Code 的记忆系统不是"一个压缩算法"，而是 **七层递进的防御体系**：

```
第 1 层（工具结果存储）→ 预防：不让大结果占满上下文
第 2 层（微压缩）      → 维护：定期清理旧数据
第 3 层（会话记忆）    → 整理：实时结构化笔记
第 4 层（全压缩）      → 急救：紧急情况下的大规模压缩
第 5 层（自动记忆提取）→ 沉淀：跨会话的知识保留
第 6 层（做梦机制）    → 巩固：后台知识整合
第 7 层（跨代理沟通）  → 协作：多 Agent 间的知识传递
```

**类比**：就像你家的清洁系统——
- 第 1 层 = 门口放个鞋架，不让脏东西进屋
- 第 2 层 = 每天扫地，清理日常垃圾
- 第 3 层 = 每周整理，把东西归位
- 第 4 层 = 大扫除，彻底清理
- 第 5-7 层 = 长期维护（下一章）

### 2. 缓存命中是架构原则，不是优化

Claude Code 的每一个设计决策都考虑缓存命中：

| 设计 | 缓存考虑 |
|------|----------|
| 状态冻结（pinCacheEdits） | 确保 Prompt 前缀字节一致 |
| 缓存编辑压缩（cache_edits） | 服务器端删除，本地不变 |
| 14 个缓存断点 | 精确控制缓存边界 |
| 粘性锁存器 | 防止模式切换导致失效 |
| 只运行主线 | 避免子代理破坏主线程缓存 |

**为什么缓存这么重要？** Anthropic API 的 Prompt Caching 可以节省 **50-90% 的 API 成本**。对于一个每天处理数千次对话的产品来说，这不是优化，是经济命脉。

### 3. Token 估算 vs 精确计数

| 方法 | 精度 | 速度 | 成本 | 适用场景 |
|------|------|------|------|----------|
| API 精确计数 | 100% | 慢（网络请求） | 免费 | 压缩决策 |
| Haiku 回退 | ~95% | 中等 | 极小（Haiku 便宜） | API 不可用时 |
| 粗糙估算 | ~80% | 极快 | 零 | 实时监控 |

Claude Code 的策略：**日常用粗糙估算，压缩决策时用 API 精确计数**。

### 4. 消息类型与 Token 权重

在 Claude Code 的消息系统中，不同类型的内容 token 消耗差异巨大：

```
系统提示词（工具定义）  ≈ 10,000-30,000 tokens
文件读取结果           ≈ 5,000-25,000 tokens/文件
工具调用               ≈ 500-2,000 tokens/次
用户消息               ≈ 100-1,000 tokens
助手回复               ≈ 500-3,000 tokens
```

这就是为什么 Claude Code 要限制文件读取的 token 数（25,000），以及为什么第 1 层要对工具结果做预算控制。

### 代码 1：`memory/token_counter.py`

```python
"""
Token 计数与估算模块。
参考 Claude Code 的 tokenEstimation.ts 实现。
"""

import json
import math
from typing import Optional


class TokenEstimator:
    """
    Token 估算器。

    提供三种精度的 token 计数：
    1. 粗糙估算（基于字符数，4 chars/token）
    2. 类型感知估算（JSON 用 2 chars/token）
    3. API 精确计数（需要 Anthropic API Key）

    Claude Code 的策略：
    - 日常监控用粗糙估算
    - 压缩决策用 API 精确计数
    """

    DEFAULT_BYTES_PER_TOKEN = 4
    DENSE_BYTES_PER_TOKEN = 2
    IMAGE_TOKEN_ESTIMATE = 2000
    DOCUMENT_TOKEN_ESTIMATE = 2000

    def __init__(self, api_key: Optional[str] = None):
        self.api_key = api_key

    def estimate(self, text: str) -> int:
        """粗糙估算：tokens ≈ len(text) / 4"""
        return math.ceil(len(text) / self.DEFAULT_BYTES_PER_TOKEN)

    def estimate_for_type(self, text: str, file_type: str = '') -> int:
        """类型感知的 token 估算"""
        ext = file_type.lower().lstrip('.')
        if ext in ('json', 'jsonl', 'jsonc'):
            return math.ceil(len(text) / self.DENSE_BYTES_PER_TOKEN)
        return self.estimate(text)

    def estimate_message(self, message: dict) -> int:
        """估算一条消息的 token 数"""
        content = message.get('content', '')
        if isinstance(content, str):
            return self.estimate(content)
        if isinstance(content, list):
            return sum(self._estimate_block(block) for block in content)
        return 0

    def estimate_messages(self, messages: list[dict]) -> int:
        """估算消息列表的总 token 数"""
        return sum(self.estimate_message(m) for m in messages)

    def _estimate_block(self, block: dict) -> int:
        """估算单个内容块的 token 数"""
        block_type = block.get('type', 'text')

        if block_type == 'text':
            return self.estimate(block.get('text', ''))

        if block_type == 'tool_use':
            name = block.get('name', '')
            inp = block.get('input', {})
            return self.estimate_for_type(
                name + json.dumps(inp, ensure_ascii=False), 'json'
            )

        if block_type == 'tool_result':
            content = block.get('content', '')
            if isinstance(content, str):
                return self.estimate(content)
            if isinstance(content, list):
                return sum(self._estimate_block(b) for b in content)
            return 0

        if block_type == 'image':
            return self.IMAGE_TOKEN_ESTIMATE
        if block_type == 'document':
            return self.DOCUMENT_TOKEN_ESTIMATE

        return self.estimate_for_type(
            json.dumps(block, ensure_ascii=False), 'json'
        )

    def count_tokens_api(self, text: str) -> Optional[int]:
        """使用 Anthropic API 精确计数 token"""
        if not self.api_key:
            return None
        try:
            from anthropic import Anthropic
            client = Anthropic(api_key=self.api_key)
            response = client.beta.messages.count_tokens(
                model="claude-3-5-sonnet-20241022",
                messages=[{"role": "user", "content": text}],
            )
            return response.input_tokens
        except Exception:
            return None

    @classmethod
    def analyze_context(cls, messages: list[dict]) -> dict:
        """分析上下文，返回详细的 token 分布"""
        estimator = cls()
        result = {
            'total_tokens': 0,
            'user_tokens': 0,
            'assistant_tokens': 0,
            'tool_tokens': 0,
            'message_count': len(messages),
        }
        for msg in messages:
            tokens = estimator.estimate_message(msg)
            result['total_tokens'] += tokens
            role = msg.get('role', '')
            if role == 'user':
                result['user_tokens'] += tokens
            elif role == 'assistant':
                result['assistant_tokens'] += tokens
            elif role == 'tool':
                result['tool_tokens'] += tokens
        return result
```

### 代码 2：`memory/tool_budget.py`（第 1 层）

```python
"""
第 1 层：工具结果预算管理。

参考 Claude Code 的 src/tools/ 工具结果预算系统。

核心功能：
1. COMPACTABLE_TOOLS 集合：哪些工具的结果需要预算管理
2. 磁盘持久化：超过阈值时完整结果写磁盘
3. 预览截断：上下文只放前 ~2KB 预览
4. 状态冻结（pinCacheEdits）：确保后续 API 调用使用相同预览
5. 远程调节：每个工具的阈值可通过配置远程调节
"""

import os
import time
import hashlib
from dataclasses import dataclass, field
from pathlib import Path
from typing import Optional


# ============================================================
# 可压缩工具集合（参考 Claude Code 的 COMPACTABLE_TOOLS）
# ============================================================

COMPACTABLE_TOOLS = frozenset([
    'FileRead',
    'Bash',
    'Grep',
    'Glob',
    'WebSearch',
    'WebFetch',
    'FileEdit',
    'FileWrite',
])


# ============================================================
# 默认阈值配置（可通过远程配置覆盖）
# ============================================================

DEFAULT_THRESHOLDS = {
    'FileRead': 25_000,    # 文件读取：25K tokens
    'Bash': 10_000,        # 命令执行：10K tokens
    'Grep': 15_000,        # 文本搜索：15K tokens
    'Glob': 5_000,         # 文件匹配：5K tokens
    'WebSearch': 8_000,    # 网络搜索：8K tokens
    'WebFetch': 20_000,    # 网页抓取：20K tokens
    'FileEdit': 2_000,     # 文件编辑：2K tokens
    'FileWrite': 2_000,    # 文件写入：2K tokens
}

DEFAULT_MAX_PREVIEW = 2048  # 预览最大字符数（约 512 tokens）


# ============================================================
# 工具结果存储
# ============================================================

@dataclass
class ToolResultRecord:
    """工具结果的完整记录"""
    tool_name: str
    tool_call_id: str
    full_result: str
    preview: str
    is_truncated: bool
    disk_path: Optional[str] = None
    timestamp: float = field(default_factory=time.time)
    token_count: int = 0

    @property
    def summary(self) -> str:
        """结果摘要"""
        if self.is_truncated:
            return (
                f"[{self.tool_name} result truncated: "
                f"full result saved to {self.disk_path or 'disk'}]"
            )
        return self.full_result


class ToolResultBudget:
    """
    工具结果预算管理器。

    核心职责：
    1. 判断工具结果是否超过阈值
    2. 超过阈值时：完整结果写磁盘，返回预览
    3. 状态冻结：一旦决定用预览，后续都用同样的预览
    4. 远程调节：支持动态调整每个工具的阈值

    类比：门口的鞋架——不让脏东西（大结果）进屋（上下文）。
    """

    def __init__(
        self,
        max_preview: int = DEFAULT_MAX_PREVIEW,
        persist_dir: str = ".agent/tool-results",
        session_id: str = "default",
        thresholds: Optional[dict[str, int]] = None,
    ):
        """
        Args:
            max_preview: 预览最大字符数（默认 2048）
            persist_dir: 磁盘持久化目录
            session_id: 会话 ID（用于隔离不同会话的结果）
            thresholds: 各工具的阈值配置（None 时使用默认值）
        """
        self.max_preview = max_preview
        self.session_id = session_id
        self.thresholds = thresholds or dict(DEFAULT_THRESHOLDS)

        # 持久化目录：tool-results/<session>/
        self.persist_dir = Path(persist_dir) / session_id
        self.persist_dir.mkdir(parents=True, exist_ok=True)

        # 状态冻结：tool_call_id -> 冻结的预览
        self._pinned_previews: dict[str, str] = {}

        # 结果记录：tool_call_id -> ToolResultRecord
        self._records: dict[str, ToolResultRecord] = {}

        # Token 估算器（延迟导入）
        self._estimator = None

    def process_result(
        self,
        tool_name: str,
        tool_call_id: str,
        result: str,
    ) -> ToolResultRecord:
        """
        处理工具结果，超过阈值时持久化到磁盘，返回预览。

        流程：
        1. 检查是否已冻结预览（状态冻结）
        2. 检查工具是否在 COMPACTABLE_TOOLS 中
        3. 检查结果是否超过阈值
        4. 超过阈值：写磁盘 + 返回预览
        5. 未超过：返回完整结果

        Args:
            tool_name: 工具名称
            tool_call_id: 工具调用 ID
            result: 工具执行的完整结果

        Returns:
            ToolResultRecord 记录
        """
        # 1. 检查状态冻结
        if tool_call_id in self._pinned_previews:
            return self._records.get(tool_call_id) or self._create_pinned_record(
                tool_name, tool_call_id, result
            )

        # 2. 检查是否需要预算管理
        if tool_name not in COMPACTABLE_TOOLS:
            record = ToolResultRecord(
                tool_name=tool_name,
                tool_call_id=tool_call_id,
                full_result=result,
                preview=result,
                is_truncated=False,
                token_count=self._estimate_tokens(result),
            )
            self._records[tool_call_id] = record
            return record

        # 3. 检查是否超过阈值
        threshold = self.thresholds.get(tool_name, DEFAULT_MAX_PREVIEW * 4)
        token_count = self._estimate_tokens(result)

        if token_count <= threshold:
            # 未超过阈值，直接返回完整结果
            record = ToolResultRecord(
                tool_name=tool_name,
                tool_call_id=tool_call_id,
                full_result=result,
                preview=result,
                is_truncated=False,
                token_count=token_count,
            )
        else:
            # 超过阈值：写磁盘 + 生成预览
            preview = self._create_preview(result)
            disk_path = self._persist_to_disk(tool_call_id, result)

            record = ToolResultRecord(
                tool_name=tool_name,
                tool_call_id=tool_call_id,
                full_result=preview,  # 上下文中只放预览
                preview=preview,
                is_truncated=True,
                disk_path=disk_path,
                token_count=self._estimate_tokens(preview),
            )

            # 4. 状态冻结：锁定预览
            self._pinned_previews[tool_call_id] = preview

        self._records[tool_call_id] = record
        return record

    def pin_preview(self, tool_call_id: str, preview: str) -> None:
        """
        冻结预览，确保后续 API 调用使用相同的预览。

        这是 Claude Code 的 pinCacheEdits 机制：
        一旦决定用预览，后续所有 API 调用都用同样的预览文本，
        确保 Prompt 前缀字节一致，最大化缓存命中率。
        """
        self._pinned_previews[tool_call_id] = preview

    def get_full_result(self, tool_call_id: str) -> Optional[str]:
        """
        从磁盘读取完整结果。

        当 Agent 在后续对话中需要完整内容时调用。
        """
        record = self._records.get(tool_call_id)
        if not record:
            return None

        if not record.is_truncated:
            return record.full_result

        if record.disk_path:
            try:
                return Path(record.disk_path).read_text(encoding='utf-8')
            except FileNotFoundError:
                return None
        return None

    def update_threshold(self, tool_name: str, new_threshold: int) -> None:
        """
        远程调节：动态调整某个工具的阈值。

        模拟 GrowthBook 功能标志的远程配置能力。
        """
        self.thresholds[tool_name] = new_threshold

    def get_stats(self) -> dict:
        """获取预算统计信息"""
        total_saved = sum(
            r.token_count for r in self._records.values() if r.is_truncated
        )
        return {
            'total_records': len(self._records),
            'truncated_records': sum(
                1 for r in self._records.values() if r.is_truncated
            ),
            'pinned_previews': len(self._pinned_previews),
            'tokens_saved_by_truncation': total_saved,
            'persist_dir': str(self.persist_dir),
        }

    # ============================================================
    # 内部方法
    # ============================================================

    def _create_preview(self, result: str) -> str:
        """创建预览：截取前 max_preview 个字符"""
        if len(result) <= self.max_preview:
            return result

        preview = result[:self.max_preview]
        # 找到最后一个完整行
        last_newline = preview.rfind('\n')
        if last_newline > self.max_preview * 0.8:
            preview = preview[:last_newline]

        preview += f"\n\n... [truncated, {len(result) - self.max_preview} more chars]"
        return preview

    def _persist_to_disk(self, tool_call_id: str, result: str) -> str:
        """将完整结果持久化到磁盘"""
        # 用 hash 生成文件名，避免冲突
        file_hash = hashlib.md5(tool_call_id.encode()).hexdigest()[:8]
        file_path = self.persist_dir / f"{file_hash}.txt"
        file_path.write_text(result, encoding='utf-8')
        return str(file_path)

    def _estimate_tokens(self, text: str) -> int:
        """估算 token 数"""
        if self._estimator is None:
            from .token_counter import TokenEstimator
            self._estimator = TokenEstimator()
        return self._estimator.estimate(text)

    def _create_pinned_record(
        self, tool_name: str, tool_call_id: str, result: str
    ) -> ToolResultRecord:
        """为已冻结的预览创建记录"""
        preview = self._pinned_previews[tool_call_id]
        record = ToolResultRecord(
            tool_name=tool_name,
            tool_call_id=tool_call_id,
            full_result=preview,
            preview=preview,
            is_truncated=True,
            token_count=self._estimate_tokens(preview),
        )
        self._records[tool_call_id] = record
        return record
```

### 代码 3：`memory/micro_compact.py`（第 2 层）

```python
"""
第 2 层：微压缩（Micro Compact）。

参考 Claude Code 的 src/services/compact/microCompact.ts。

核心功能：
1. 基于时间的清理：超过 60 分钟的旧工具结果替换为占位符
2. 缓存编辑压缩（cache_edits）：服务器端删除，本地不变
3. API 级上下文管理：标记可跳过的消息

关键设计：只运行主线。子代理不能修改全局缓存状态。
"""

import time
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional


# ============================================================
# 消息标记
# ============================================================

class CompactAction(Enum):
    """微压缩的动作类型"""
    KEEP = "keep"              # 保留
    OMIT = "omit"              # 跳过（服务器端删除）
    REPLACE = "replace"        # 替换为占位符


@dataclass
class MicroCompactDecision:
    """单条消息的微压缩决策"""
    message_index: int
    action: CompactAction
    reason: str = ""
    replacement: Optional[str] = None


# ============================================================
# 微压缩器
# ============================================================

class MicroCompact:
    """
    微压缩 — 每轮对话前的轻量清理。

    三种机制：
    1. 基于时间的清理：距离上次助手消息超过阈值，清理旧工具结果
    2. 缓存编辑压缩：标记可删除的旧工具结果（不修改本地消息）
    3. API 级上下文管理：使用 context_management 参数

    类比：日常保洁——定期清理已经进屋的垃圾。

    关键约束：只运行主线。子代理不能修改全局缓存状态。
    """

    def __init__(
        self,
        time_threshold_minutes: int = 60,
        max_omit_messages: int = 50,
    ):
        """
        Args:
            time_threshold_minutes: 时间阈值（分钟），超过此时间的旧消息可清理
            max_omit_messages: 最多可标记为跳过的消息数
        """
        self.time_threshold = time_threshold_minutes * 60  # 转为秒
        self.max_omit_messages = max_omit_messages

        # 缓存编辑状态：记录哪些消息已被标记为可跳过
        self._omitted_indices: set[int] = set()

        # 主线保护：确保只在主线运行
        self._is_main_branch = True

    def evaluate(
        self,
        messages: list[dict],
        current_time: Optional[float] = None,
    ) -> list[MicroCompactDecision]:
        """
        评估消息列表，生成微压缩决策。

        组合三种机制：
        1. 时间清理
        2. 缓存编辑
        3. API 上下文管理

        Args:
            messages: 消息列表，每条消息应包含 'timestamp' 字段
            current_time: 当前时间（用于测试）

        Returns:
            微压缩决策列表
        """
        if not self._is_main_branch:
            # 非主线：不做缓存编辑，避免破坏主线程缓存
            return self._evaluate_time_based_only(messages, current_time)

        decisions = []
        decisions.extend(
            self._evaluate_time_based(messages, current_time)
        )
        decisions.extend(
            self._evaluate_cache_based(messages)
        )

        # 去重：同一条消息只保留最激进的决策
        return self._deduplicate_decisions(decisions)

    def apply_omissions(
        self,
        messages: list[dict],
        decisions: list[MicroCompactDecision],
    ) -> tuple[list[dict], dict]:
        """
        应用微压缩决策，返回 API 请求格式。

        关键：本地消息列表不变，只生成 API 请求的跳过标记。

        Returns:
            (api_messages, context_management)
            - api_messages: 发送给 API 的消息（本地不变）
            - context_management: 上下文管理参数
        """
        omit_indices = set()
        for decision in decisions:
            if decision.action == CompactAction.OMIT:
                omit_indices.add(decision.message_index)
            elif decision.action == CompactAction.REPLACE:
                # 替换为占位符（修改消息内容）
                if decision.replacement:
                    msg = messages[decision.message_index]
                    if isinstance(msg.get('content'), str):
                        msg['content'] = decision.replacement

        # 构建 context_management 参数
        context_management = {
            'omitted_indices': sorted(omit_indices),
            'omit_count': len(omit_indices),
        }

        return messages, context_management

    def set_branch_mode(self, is_main: bool) -> None:
        """
        设置分支模式。

        非主线时禁用缓存编辑，避免破坏主线程缓存。
        """
        self._is_main_branch = is_main

    # ============================================================
    # 内部方法
    # ============================================================

    def _evaluate_time_based(
        self,
        messages: list[dict],
        current_time: Optional[float] = None,
    ) -> list[MicroCompactDecision]:
        """
        基于时间的清理：超过阈值的旧工具结果替换为占位符。

        规则：
        - 找到最后一条助手消息的时间
        - 如果距离现在超过阈值，标记旧的工具结果为 REPLACE
        """
        now = current_time or time.time()
        decisions = []

        # 找到最后一条助手消息的时间
        last_assistant_time = None
        for i, msg in enumerate(messages):
            if msg.get('role') == 'assistant':
                last_assistant_time = msg.get('timestamp', now)

        if last_assistant_time is None:
            return decisions

        # 检查时间差
        time_diff = now - last_assistant_time
        if time_diff < self.time_threshold:
            return decisions

        # 标记旧的工具结果
        for i, msg in enumerate(messages):
            msg_time = msg.get('timestamp', now)
            if msg_time >= last_assistant_time:
                break  # 只处理最后一条助手消息之前的内容

            # 检查是否是工具结果
            content = msg.get('content', '')
            is_tool_result = False
            if isinstance(content, list):
                is_tool_result = any(
                    block.get('type') == 'tool_result'
                    for block in content
                )
            elif isinstance(content, dict):
                is_tool_result = content.get('type') == 'tool_result'

            if is_tool_result and i not in self._omitted_indices:
                decisions.append(MicroCompactDecision(
                    message_index=i,
                    action=CompactAction.REPLACE,
                    reason=f"time_based: {time_diff:.0f}s > {self.time_threshold:.0f}s",
                    replacement='[tool result omitted: time-based cleanup]',
                ))

        return decisions

    def _evaluate_cache_based(
        self,
        messages: list[dict],
    ) -> list[MicroCompactDecision]:
        """
        缓存编辑压缩：标记可删除的旧工具结果（不修改本地消息）。

        规则：
        - 标记最早的工具结果为 OMIT
        - 本地消息不变，通过 API 的 context_management 参数跳过
        - 最多标记 max_omit_messages 条
        """
        decisions = []
        omit_count = 0

        for i, msg in enumerate(messages):
            if omit_count >= self.max_omit_messages:
                break

            if i in self._omitted_indices:
                continue  # 已标记过

            content = msg.get('content', '')
            is_tool_result = False
            if isinstance(content, list):
                is_tool_result = any(
                    block.get('type') == 'tool_result'
                    for block in content
                )

            if is_tool_result:
                decisions.append(MicroCompactDecision(
                    message_index=i,
                    action=CompactAction.OMIT,
                    reason="cache_edits: server-side removal",
                ))
                self._omitted_indices.add(i)
                omit_count += 1

        return decisions

    def _evaluate_time_based_only(
        self,
        messages: list[dict],
        current_time: Optional[float] = None,
    ) -> list[MicroCompactDecision]:
        """非主线模式：只做时间清理，不做缓存编辑"""
        return self._evaluate_time_based(messages, current_time)

    def _deduplicate_decisions(
        self,
        decisions: list[MicroCompactDecision],
    ) -> list[MicroCompactDecision]:
        """去重：同一条消息只保留最激进的决策"""
        best: dict[int, MicroCompactDecision] = {}
        priority = {
            CompactAction.OMIT: 2,
            CompactAction.REPLACE: 1,
            CompactAction.KEEP: 0,
        }

        for d in decisions:
            existing = best.get(d.message_index)
            if existing is None or priority[d.action] > priority[existing.action]:
                best[d.message_index] = d

        return sorted(best.values(), key=lambda d: d.message_index)
```

### 代码 4：`memory/session_memory.py`（第 3 层）

```python
"""
第 3 层：会话记忆（Session Memory）。

参考 Claude Code 的会话记忆系统。

核心功能：
1. 触发条件：Token 增长达到阈值 +（工具调用次数达标 或 上轮无工具调用）
2. 零 API 压缩策略：需要压缩时直接注入现成的会话记忆摘要
3. Prompt Cache 保存策略：14 个缓存断点，粘性锁存器管理
"""

import time
from dataclasses import dataclass, field
from typing import Optional


# ============================================================
# 配置参数（参考 Claude Code 源码）
# ============================================================

@dataclass
class SessionMemoryConfig:
    """会话记忆配置"""
    minimum_message_tokens_to_init: int = 5000     # 消息总 token 达到 5000 才初始化
    minimum_tokens_between_update: int = 3000      # 两次更新之间至少增长 3000 tokens
    tool_calls_between_updates: int = 10           # 或者工具调用次数达到 10 次
    max_summary_tokens: int = 2000                 # 摘要最大 token 数
    cache_breakpoints: int = 14                    # 缓存断点数量


# ============================================================
# 缓存断点
# ============================================================

class CacheBreakpoint:
    """
    Prompt 缓存断点。

    Claude Code 维护 14 个缓存断点，精确控制缓存边界。
    每个断点标记 Prompt 中一个逻辑段的结束位置。
    """

    # 预定义的断点名称
    NAMES = [
        "system_prompt_end",          # 0: 系统提示词结束
        "tool_definitions_end",       # 1: 工具定义结束
        "skills_end",                 # 2: 技能注入结束
        "session_memory_summary",     # 3: 会话记忆摘要
        "compact_boundary",           # 4: 压缩边界标记
        "attachments_start",          # 5: 附件开始
        "attachments_end",            # 6: 附件结束
        "conversation_start",         # 7: 对话开始
        "user_message_1",             # 8: 第一条用户消息
        "assistant_message_1",        # 9: 第一条助手消息
        "mid_conversation",           # 10: 对话中段
        "recent_messages_start",      # 11: 最近消息开始
        "last_user_message",          # 12: 最后一条用户消息
        "end",                        # 13: 结束
    ]

    def __init__(self, index: int, name: str = ""):
        self.index = index
        self.name = name or (self.NAMES[index] if index < len(self.NAMES) else f"breakpoint_{index}")
        self._sticky_content_hash: Optional[str] = None  # 粘性锁存器

    def is_cache_hit(self, current_content_hash: str) -> bool:
        """
        检查缓存是否命中。

        使用粘性锁存器：即使内容变了，如果变化不大，仍然认为缓存命中。
        """
        if self._sticky_content_hash is None:
            self._sticky_content_hash = current_content_hash
            return True

        if self._sticky_content_hash == current_content_hash:
            return True

        # 粘性逻辑：如果内容变化小于阈值，仍然认为命中
        # 这里简化为直接比较，实际实现可以用编辑距离
        return False

    def update(self, content_hash: str) -> None:
        """更新锁存器"""
        self._sticky_content_hash = content_hash


# ============================================================
# 会话记忆
# ============================================================

@dataclass
class SessionMemoryEntry:
    """会话记忆条目"""
    content: str
    token_count: int
    created_at: float = field(default_factory=time.time)
    updated_at: float = field(default_factory=time.time)
    category: str = "general"  # general, technical, decision, action


class SessionMemory:
    """
    会话记忆管理器 — 第 3 层。

    核心职责：
    1. 持续监控对话，在对话过程中实时更新结构化笔记
    2. 触发条件：Token 增长 + 工具调用次数
    3. 零 API 压缩：需要压缩时直接注入现成摘要
    4. 管理 14 个缓存断点，用粘性锁存器保护缓存命中

    类比：实时笔记员——在会议过程中持续做笔记，
    需要总结时直接拿出已有的笔记，不用重新整理。
    """

    def __init__(
        self,
        config: Optional[SessionMemoryConfig] = None,
    ):
        self.config = config or SessionMemoryConfig()

        # 记忆条目
        self.entries: list[SessionMemoryEntry] = []

        # 追踪状态
        self._total_tokens_at_last_update = 0
        self._tool_calls_since_last_update = 0
        self._total_tool_calls = 0
        self._initialized = False

        # 缓存断点
        self.cache_breakpoints = [
            CacheBreakpoint(i, name)
            for i, name in enumerate(CacheBreakpoint.NAMES)
        ]

        # 当前摘要
        self._current_summary: str = ""

    def record_tool_call(self) -> None:
        """记录一次工具调用"""
        self._total_tool_calls += 1
        self._tool_calls_since_last_update += 1

    def should_update(self, current_total_tokens: int) -> bool:
        """
        检查是否应该更新会话记忆。

        触发条件（参考 Claude Code 源码）：
        1. 消息总 token 达到 minimum_message_tokens_to_init
        2. 且满足以下任一条件：
           a. Token 增长达到 minimum_tokens_between_update
           b. 工具调用次数达到 tool_calls_between_updates
        """
        if not self._initialized:
            return current_total_tokens >= self.config.minimum_message_tokens_to_init

        token_growth = current_total_tokens - self._total_tokens_at_last_update

        return (
            token_growth >= self.config.minimum_tokens_between_update
            or self._tool_calls_since_last_update >= self.config.tool_calls_between_updates
        )

    def update(self, summary: str, current_total_tokens: int) -> None:
        """
        更新会话记忆。

        Args:
            summary: 新的会话摘要（由 LLM 生成或手动更新）
            current_total_tokens: 当前总 token 数
        """
        self._current_summary = summary
        self._total_tokens_at_last_update = current_total_tokens
        self._tool_calls_since_last_update = 0
        self._initialized = True

        entry = SessionMemoryEntry(
            content=summary,
            token_count=self._estimate_tokens(summary),
            category="general",
        )
        self.entries.append(entry)

        # 更新缓存断点（会话记忆摘要断点）
        summary_bp = self.cache_breakpoints[3]  # session_memory_summary
        summary_bp.update(self._hash_content(summary))

    def get_injection_prompt(self) -> str:
        """
        获取注入提示词。

        当需要压缩时，直接返回现成的会话记忆摘要，
        不需要额外调用 LLM（零 API 策略）。
        """
        if not self._current_summary:
            return ""

        return (
            f"[Session Memory — Auto-generated summary]\n"
            f"{self._current_summary}\n"
            f"[End of Session Memory]\n"
        )

    def get_cache_status(self) -> dict:
        """获取缓存断点状态"""
        return {
            'breakpoints': len(self.cache_breakpoints),
            'entries': len(self.entries),
            'summary_tokens': self._estimate_tokens(self._current_summary),
            'initialized': self._initialized,
            'total_tool_calls': self._total_tool_calls,
        }

    def add_entry(
        self,
        content: str,
        category: str = "general",
    ) -> None:
        """手动添加记忆条目"""
        entry = SessionMemoryEntry(
            content=content,
            token_count=self._estimate_tokens(content),
            category=category,
        )
        self.entries.append(entry)

    def get_entries_by_category(self, category: str) -> list[SessionMemoryEntry]:
        """按类别获取记忆条目"""
        return [e for e in self.entries if e.category == category]

    # ============================================================
    # 内部方法
    # ============================================================

    def _estimate_tokens(self, text: str) -> int:
        """估算 token 数"""
        return max(1, len(text) // 4)

    def _hash_content(self, content: str) -> str:
        """内容哈希（用于缓存断点比较）"""
        import hashlib
        return hashlib.md5(content.encode()).hexdigest()[:8]
```

```python
"""
短期记忆管理 —— 上下文窗口与消息压缩（整合层）。

整合第 1-3 层：
- 第 1 层：ToolResultBudget（工具结果预算）
- 第 2 层：MicroCompact（微压缩）
- 第 3 层：SessionMemory（会话记忆）

以及原有的上下文窗口管理。
"""

import time
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional

from .token_counter import TokenEstimator
from .tool_budget import ToolResultBudget
from .micro_compact import MicroCompact
from .session_memory import SessionMemory


class MessageType(Enum):
    USER = "user"
    ASSISTANT = "assistant"
    TOOL_RESULT = "tool_result"
    SYSTEM = "system"
    COMPACT_BOUNDARY = "compact_boundary"
    ATTACHMENT = "attachment"


@dataclass
class Message:
    """统一的消息表示"""
    role: str
    content: str | list[dict]
    msg_type: MessageType = MessageType.USER
    timestamp: float = field(default_factory=time.time)
    metadata: dict = field(default_factory=dict)

    def to_dict(self) -> dict:
        return {
            'role': self.role,
            'content': self.content,
            'type': self.msg_type.value,
            'timestamp': self.timestamp,
            'metadata': self.metadata,
        }


@dataclass
class CompactBoundary:
    """压缩边界标记"""
    trigger: str
    pre_compact_tokens: int
    post_compact_tokens: int
    messages_summarized: int
    timestamp: float = field(default_factory=time.time)

    def to_message(self) -> Message:
        return Message(
            role='system',
            content=(
                f"[Conversation compacted: {self.messages_summarized} messages "
                f"summarized. Pre-compact: {self.pre_compact_tokens} tokens, "
                f"Post-compact: {self.post_compact_tokens} tokens. "
                f"Trigger: {self.trigger}]"
            ),
            msg_type=MessageType.COMPACT_BOUNDARY,
            timestamp=self.timestamp,
        )


class ContextWindow:
    """
    上下文窗口管理器（整合第 1-3 层）。

    七层记忆架构的前三层在这里协同工作：
    - 添加工具结果时 → 第 1 层（ToolResultBudget）介入
    - 每轮对话前 → 第 2 层（MicroCompact）清理
    - Token 增长时 → 第 3 层（SessionMemory）更新
    - 超过阈值时 → 触发全压缩（第 4 层，下一章）
    """

    def __init__(
        self,
        max_tokens: int = 150_000,
        compact_threshold: float = 0.75,
        keep_recent: int = 10,
        estimator: Optional[TokenEstimator] = None,
        compact_callback=None,
        session_id: str = "default",
    ):
        self.max_tokens = max_tokens
        self.compact_threshold = compact_threshold
        self.keep_recent = keep_recent
        self.estimator = estimator or TokenEstimator()
        self.compact_callback = compact_callback

        self.messages: list[Message] = []
        self.compact_count = 0
        self.total_tokens = 0

        # 第 1 层：工具结果预算
        self.tool_budget = ToolResultBudget(session_id=session_id)

        # 第 2 层：微压缩
        self.micro_compact = MicroCompact()

        # 第 3 层：会话记忆
        self.session_memory = SessionMemory()

    def add(self, message: Message) -> None:
        """添加一条消息"""
        self.messages.append(message)
        self.total_tokens = self._estimate_total()

    def add_user(self, content: str) -> None:
        """添加用户消息"""
        self.add(Message(
            role='user',
            content=content,
            msg_type=MessageType.USER,
        ))

    def add_assistant(self, content: str) -> None:
        """添加助手消息"""
        self.add(Message(
            role='assistant',
            content=content,
            msg_type=MessageType.ASSISTANT,
        ))

    def add_tool_result(
        self,
        tool_name: str,
        tool_call_id: str,
        result: str,
    ) -> str:
        """
        添加工具调用结果（经过第 1 层预算管理）。

        Returns:
            实际放入上下文的文本（可能是预览）
        """
        record = self.tool_budget.process_result(
            tool_name, tool_call_id, result
        )

        self.add(Message(
            role='user',
            content=[{
                'type': 'tool_result',
                'tool_name': tool_name,
                'content': record.full_result,  # 可能是预览
            }],
            msg_type=MessageType.TOOL_RESULT,
        ))

        # 记录工具调用（第 3 层追踪）
        self.session_memory.record_tool_call()

        return record.preview if record.is_truncated else result

    def pre_turn_maintenance(self) -> dict:
        """
        每轮对话前的维护操作。

        执行第 2 层微压缩，检查第 3 层是否需要更新。

        Returns:
            维护操作的结果摘要
        """
        result = {
            'micro_compact': None,
            'session_memory_updated': False,
        }

        # 第 2 层：微压缩
        api_messages = [m.to_dict() for m in self.messages]
        decisions = self.micro_compact.evaluate(api_messages)
        if decisions:
            api_messages, ctx_mgmt = self.micro_compact.apply_omissions(
                api_messages, decisions
            )
            result['micro_compact'] = {
                'omitted': ctx_mgmt['omit_count'],
                'decisions': len(decisions),
            }

        # 第 3 层：检查是否需要更新会话记忆
        if self.session_memory.should_update(self.total_tokens):
            # 这里应该调用 LLM 生成摘要
            # 简化处理：标记需要更新
            result['session_memory_updated'] = True

        return result

    def needs_compact(self) -> bool:
        """检查是否需要全压缩（第 4 层）"""
        threshold = self.max_tokens * self.compact_threshold
        return self.total_tokens > threshold

    def get_usage_ratio(self) -> float:
        """获取当前 token 使用率"""
        return self.total_tokens / self.max_tokens

    def compact(self) -> Optional[CompactBoundary]:
        """
        执行上下文压缩（第 4 层，简化版）。

        注意：完整的全压缩实现在下一章。
        """
        if not self.needs_compact():
            return None
        if not self.compact_callback:
            raise RuntimeError("No compact_callback set.")

        pre_tokens = self.total_tokens
        messages_to_summarize = self.messages[:-self.keep_recent]
        messages_to_keep = self.messages[-self.keep_recent:]

        if not messages_to_summarize:
            return None

        summary = self.compact_callback(messages_to_summarize)

        summary_message = Message(
            role='user',
            content=f"[Conversation Summary]\n{summary}",
            msg_type=MessageType.USER,
            metadata={'is_compact_summary': True},
        )

        boundary = CompactBoundary(
            trigger='auto',
            pre_compact_tokens=pre_tokens,
            post_compact_tokens=0,
            messages_summarized=len(messages_to_summarize),
        )

        self.messages = [
            summary_message,
            boundary.to_message(),
            *messages_to_keep,
        ]

        self.compact_count += 1
        self.total_tokens = self._estimate_total()
        boundary.post_compact_tokens = self.total_tokens

        return boundary

    def get_api_messages(self) -> list[dict]:
        """获取发送给 API 的消息格式"""
        api_messages = []
        for msg in self.messages:
            if msg.msg_type in (MessageType.SYSTEM, MessageType.COMPACT_BOUNDARY):
                continue
            api_messages.append({
                'role': msg.role,
                'content': msg.content,
            })
        return api_messages

    def get_status(self) -> dict:
        """获取完整的上下文状态（包含三层信息）"""
        return {
            'total_tokens': self.total_tokens,
            'max_tokens': self.max_tokens,
            'usage_ratio': f"{self.get_usage_ratio():.1%}",
            'message_count': len(self.messages),
            'compact_count': self.compact_count,
            'needs_compact': self.needs_compact(),
            # 第 1 层状态
            'tool_budget': self.tool_budget.get_stats(),
            # 第 3 层状态
            'session_memory': self.session_memory.get_cache_status(),
        }

    def _estimate_total(self) -> int:
        """估算当前消息的总 token 数"""
        return self.estimator.estimate_messages(self.messages)

    def clear(self) -> None:
        """清空所有消息"""
        self.messages.clear()
        self.total_tokens = 0
        self.compact_count = 0


# ============================================================
# 媒体剥离（保留原有功能）
# ============================================================

def strip_media_blocks(content: str | list[dict]) -> str | list[dict]:
    """
    剥离图片和文档 block，替换为文本标记。
    参考 Claude Code 的 stripImagesFromMessages。
    """
    if isinstance(content, str):
        return content

    result = []
    for block in content:
        if block.get('type') == 'image':
            result.append({'type': 'text', 'text': '[image]'})
        elif block.get('type') == 'document':
            result.append({'type': 'text', 'text': '[document]'})
        else:
            result.append(block)

    return result
```

## 📝 本章小结

本章实现了 AI Agent 短期记忆的前三层防御体系：

| 层级 | 组件 | 作用 | 关键设计 |
|------|------|------|----------|
| **第 1 层** | ToolResultBudget | 工具结果预算 | COMPACTABLE_TOOLS、磁盘持久化、状态冻结、远程调节 |
| **第 2 层** | MicroCompact | 微压缩 | 时间清理、缓存编辑压缩、API 级上下文管理、只运行主线 |
| **第 3 层** | SessionMemory | 会话记忆 | 渐进式触发、零 API 压缩、14 个缓存断点、粘性锁存器 |
| **整合层** | ContextWindow | 上下文窗口 | 三层协同、自动压缩阈值、消息生命周期 |

Claude Code 的核心洞察：

1. **预防优于治疗**：第 1 层不让大结果进上下文，比进了再压缩更高效
2. **缓存命中是架构原则**：状态冻结、缓存编辑、粘性锁存器——每个设计都服务于缓存
3. **零 API 策略**：会话记忆在对话过程中持续更新，压缩时直接注入，不需要额外调用 LLM
4. **只运行主线**：子代理不能修改全局缓存状态，避免破坏主线程的缓存编辑
5. **远程调节**：阈值不是硬编码的，通过功能标志远程调整，无需发布新版本

## 🏋️ 课后练习

### 练习 1：实现自适应压缩阈值
> **题目**：当前压缩阈值是固定的（75%）。实现一个自适应阈值：当对话中工具调用频繁时降低阈值（提前压缩），当对话简单时提高阈值。
>
> **答案提示**：统计最近 10 条消息中 tool_result 的比例。比例 > 50% 时用 0.6 阈值，比例 < 20% 时用 0.85 阈值。

### 练习 2：实现消息优先级排序
> **题目**：不是所有消息都同等重要。实现消息优先级评分，压缩时优先删除低优先级消息。
>
> **答案提示**：优先级规则：包含文件路径的消息 > 纯文本消息；工具结果 > 普通对话；包含代码块的消息 > 闲聊。用分数排序，从低分开始删除。

### 练习 3：实现增量压缩
> **题目**：Claude Code 支持 partial compact（部分压缩），只压缩对话的某一部分。实现 `partial_compact(pivot_index, direction)` 方法。
>
> **答案提示**：`direction='from'` 压缩 pivot 之后的消息，`direction='up_to'` 压缩 pivot 之前的消息。保留另一侧的原始消息不变。

### 练习 4：实现 Token 预算分配器
> **题目**：实现一个 `TokenBudget` 类，管理不同组件的 token 分配：系统提示词、工具定义、消息历史、工具结果各分配多少。
>
> **答案提示**：总预算 200K。系统提示 20K，工具定义 30K，消息历史 100K，工具结果 50K。当某部分超限时，按比例压缩其他部分或触发压缩。

### 练习 5：实现缓存断点可视化
> **题目**：为 14 个缓存断点实现一个可视化函数，显示每个断点的缓存命中状态和内容哈希。
>
> **答案提示**：遍历 `cache_breakpoints`，对每个断点调用 `is_cache_hit()`，输出表格格式的状态报告。
