## 🧠 核心概念

### 什么是工具调用？

在第 1 章，我们的 Agent 只能"说话"——接收文本，回复文本。但真正的 Agent 需要能**做事**：运行命令、读写文件、搜索网络……

工具调用（Tool Calling）让 LLM 不仅能生成文本回复，还能**请求执行外部操作**。

```
用户: "帮我看看当前目录下有什么文件"

LLM 思考: 我需要执行 ls 命令
LLM 输出: {
    "tool_calls": [{
        "name": "bash",
        "arguments": {"command": "ls -la"}
    }]
}

系统执行: ls -la → 返回结果

LLM 收到结果: "当前目录下有以下文件：..."
```

### 工具调用的工作流程

```
┌──────────┐    用户输入     ┌──────────┐
│  用户     │ ────────────► │  Agent    │
│          │               │  (大脑)   │
│          │ ◄──────────── │          │
└──────────┘    最终回复    └────┬─────┘
                                 │
                    ┌────────────┼────────────┐
                    │ tool_call │             │
                    ▼           │             │
              ┌──────────┐     │      ┌──────────┐
              │  工具     │     │      │  工具     │
              │  (手)    │     │      │  (脚)    │
              │  bash    │     │      │  read    │
              └──────────┘     │      └──────────┘
                               │
                          更多工具...
```

**核心循环**（Agent 的灵魂）：

1. 用户发消息 → Agent 发给 LLM
2. LLM 决定调用工具 → 返回工具调用请求
3. Agent 执行工具 → 把结果喂回给 LLM
4. LLM 拿到结果 → 可能继续调用工具，也可能直接回复
5. 直到 LLM 不再调用工具 → 循环结束，返回最终回复

### 工具的定义格式

主流 LLM 提供商都支持工具调用，格式略有不同：

| 提供商 | 格式特点 | 关键字段 |
|--------|----------|----------|
| OpenAI | `type: "function"` 包裹 | `function.name`, `function.parameters` |
| Anthropic | 直接定义 | `name`, `input_schema` |
| 兼容层 | 两者都支持 | 我们实现 `to_openai_format()` 和 `to_anthropic_format()` |

**OpenAI 格式示例**：
```python
{
    "type": "function",
    "function": {
        "name": "bash",
        "description": "执行 Shell 命令",
        "parameters": {
            "type": "object",
            "properties": {
                "command": {"type": "string", "description": "要执行的命令"}
            },
            "required": ["command"]
        }
    }
}
```

### 第一步：创建 tools 目录和基类

创建 `src/tools/__init__.py`（空文件即可）和 `src/tools/base.py`：

```python
"""
工具系统 —— 让 AI 有手有脚
从零手写 AI Agent 课程 · 第 2 章
"""

from __future__ import annotations

from abc import ABC, abstractmethod
from typing import Any


class Tool(ABC):
    """
    工具基类。所有具体工具都必须继承此类。
    
    每个工具需要定义：
    - name: 工具名称（LLM 调用时使用的标识）
    - description: 工具描述（告诉 LLM 这个工具能做什么）
    - parameters: 参数 schema（JSON Schema 格式，告诉 LLM 需要什么参数）
    - execute(): 执行逻辑（真正做事的地方）
    """
    
    name: str = ""
    description: str = ""
    
    @property
    @abstractmethod
    def parameters(self) -> dict:
        """
        返回工具的参数 schema（JSON Schema 格式）。
        
        这个 schema 告诉 LLM：
        - 需要哪些参数
        - 每个参数的类型
        - 哪些参数是必填的
        
        示例：
        {
            "type": "object",
            "properties": {
                "command": {
                    "type": "string",
                    "description": "要执行的 Shell 命令"
                }
            },
            "required": ["command"]
        }
        """
        pass
    
    @abstractmethod
    def execute(self, **kwargs: Any) -> str:
        """
        执行工具，返回结果字符串。
        
        Args:
            **kwargs: 工具参数（由 LLM 提供）
            
        Returns:
            执行结果（字符串格式，会被送回给 LLM）
        """
        pass
    
    def to_openai_format(self) -> dict:
        """转换为 OpenAI 工具格式"""
        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": self.parameters,
            }
        }
    
    def to_anthropic_format(self) -> dict:
        """转换为 Anthropic 工具格式"""
        return {
            "name": self.name,
            "description": self.description,
            "input_schema": self.parameters,
        }
```

### 第二步：实现 BashTool

创建 `src/tools/bash_tool.py`：

```python
"""
BashTool —— 执行 Shell 命令
从零手写 AI Agent 课程 · 第 2 章
"""

from __future__ import annotations

import subprocess
from typing import Any

from tools.base import Tool


class BashTool(Tool):
    """
    执行 Shell 命令的工具。
    
    这是 Agent 最强大的工具之一——通过它，Agent 可以：
    - 查看文件和目录（ls, cat, find）
    - 运行程序（python, node, gcc）
    - 安装依赖（pip install, npm install）
    - 执行系统操作（git, curl, grep）
    
    ⚠️ 注意：BashTool 有安全风险！后续章节会加入权限控制。
    """
    
    name = "bash"
    description = (
        "执行 Shell 命令并返回输出结果。"
        "适用于查看文件、运行程序、安装依赖、执行系统操作等。"
    )
    
    # 最大输出长度，防止大输出撑爆上下文
    MAX_OUTPUT_LENGTH = 10000
    
    @property
    def parameters(self) -> dict:
        return {
            "type": "object",
            "properties": {
                "command": {
                    "type": "string",
                    "description": "要执行的 Shell 命令",
                }
            },
            "required": ["command"],
        }
    
    def execute(
        self,
        command: str,
        timeout: int = 30,
        **kwargs: Any,
    ) -> str:
        """
        执行 Shell 命令。
        
        Args:
            command: Shell 命令
            timeout: 超时时间（秒），默认 30 秒
            
        Returns:
            命令输出（stdout + stderr + 退出码）
        """
        try:
            result = subprocess.run(
                command,
                shell=True,
                capture_output=True,
                text=True,
                timeout=timeout,
            )
            
            output_parts = []
            
            # 收集 stdout
            if result.stdout:
                output_parts.append(result.stdout)
            
            # 收集 stderr
            if result.stderr:
                output_parts.append(f"\n[stderr]\n{result.stderr}")
            
            # 非零退出码
            if result.returncode != 0:
                output_parts.append(f"\n[退出码: {result.returncode}]")
            
            output = "".join(output_parts)
            
            # 截断过长的输出
            if len(output) > self.MAX_OUTPUT_LENGTH:
                output = (
                    output[:self.MAX_OUTPUT_LENGTH]
                    + "\n...（输出过长，已截断。请使用更具体的命令。）"
                )
            
            return output
            
        except subprocess.TimeoutExpired:
            return f"[错误] 命令执行超时（{timeout}秒）。请使用 timeout 参数增加超时时间。"
        except Exception as e:
            return f"[错误] {type(e).__name__}: {e}"
```

### 第三步：更新 Agent 类（⚠️ 重要变更）

⚠️ **以下代码需要更新到已有的 `src/agent.py`**。相比第 1 章的版本，主要变更：

1. 新增 `tools` 参数和 `tool_map` 字典
2. `chat()` 方法加入工具调用循环（`while True` + `tool_calls` 处理）
3. 新增 `tool_defs` 构建逻辑

```python
"""
AI Agent —— 支持工具调用
从零手写 AI Agent 课程 · 第 2 章

⚠️ 这是第 1 章 agent.py 的升级版。
   主要变更：
   - 新增 tools 参数（工具列表）
   - chat() 方法加入工具调用循环
   - 支持 OpenAI 格式的 tool_calls
"""

from __future__ import annotations

import json
import os
from typing import Optional

from openai import OpenAI
from tools.base import Tool


class Agent:
    """
    支持工具调用的 AI Agent。
    
    核心循环：
    1. 用户发消息 → 发给 LLM
    2. LLM 可能要求调用工具 → 执行工具 → 结果喂回 LLM
    3. 重复直到 LLM 不再调用工具 → 返回最终回复
    """
    
    # 最大工具调用次数，防止无限循环
    MAX_TOOL_CALLS = 10
    
    def __init__(
        self,
        system_prompt: str = "你是一个有用的 AI 助手。",
        model: str = "gpt-4o-mini",
        tools: Optional[list[Tool]] = None,
        api_key: Optional[str] = None,
    ):
        # 初始化 LLM 客户端
        self.client = OpenAI(api_key=api_key or os.environ.get("OPENAI_API_KEY"))
        self.model = model
        
        # 消息历史（初始包含 system prompt）
        self.messages = [
            {"role": "system", "content": system_prompt}
        ]
        
        # 注册工具
        self.tools = tools or []
        # 工具名称 → 工具实例的映射，方便快速查找
        self.tool_map = {tool.name: tool for tool in self.tools}
    
    def chat(self, user_input: str) -> str:
        """
        与 Agent 对话（支持工具调用）。
        
        Args:
            user_input: 用户输入
            
        Returns:
            Agent 的最终回复
        """
        # 添加用户消息
        self.messages.append({"role": "user", "content": user_input})
        
        # 构建工具定义（告诉 LLM 有哪些工具可用）
        tool_defs = [t.to_openai_format() for t in self.tools] if self.tools else None
        
        # 🔄 核心循环：LLM → 工具调用 → 执行 → 结果回传 → LLM → ...
        tool_call_count = 0
        
        while True:
            # 调用 LLM
            response = self.client.chat.completions.create(
                model=self.model,
                messages=self.messages,
                tools=tool_defs,
            )
            
            choice = response.choices[0]
            message = choice.message
            
            # 情况 1：LLM 要调用工具
            if message.tool_calls:
                tool_call_count += 1
                
                # 安全检查：防止无限循环
                if tool_call_count > self.MAX_TOOL_CALLS:
                    self.messages.append({
                        "role": "system",
                        "content": (
                            f"[警告] 工具调用次数已达上限（{self.MAX_TOOL_CALLS}次），"
                            "请停止调用工具并给出最终回复。"
                        )
                    })
                    continue
                
                # 添加工具调用请求到历史
                self.messages.append(message)
                
                # 执行每个工具调用
                for tool_call in message.tool_calls:
                    tool_name = tool_call.function.name
                    tool_args = json.loads(tool_call.function.arguments)
                    
                    print(f"  🔧 调用工具: {tool_name}({tool_args})")
                    
                    if tool_name in self.tool_map:
                        result = self.tool_map[tool_name].execute(**tool_args)
                    else:
                        result = f"[错误] 未知工具: {tool_name}"
                    
                    # 添加工具结果到历史
                    self.messages.append({
                        "role": "tool",
                        "tool_call_id": tool_call.id,
                        "content": result,
                    })
                
                # 继续循环，让 LLM 处理工具结果
                continue
            
            # 情况 2：LLM 直接回复（不再调用工具）
            reply = message.content or ""
            self.messages.append({"role": "assistant", "content": reply})
            return reply
    
    def reset(self):
        """重置对话历史，保留 system prompt"""
        self.messages = [self.messages[0]]


# === 测试运行 ===
if __name__ == "__main__":
    # 导入工具
    from tools.bash_tool import BashTool
    
    agent = Agent(
        system_prompt=(
            "你是一个有用的 AI 助手，可以执行 Shell 命令。"
            "当用户要求查看文件、运行程序、检查系统时，使用 bash 工具。"
            "回答要简洁。"
        ),
        model="gpt-4o-mini",
        tools=[BashTool()],
    )
    
    print("🤖 Agent 已启动！（输入 'quit' 退出）\n")
    
    while True:
        try:
            user_input = input("你: ").strip()
        except (EOFError, KeyboardInterrupt):
            print("\n👋 再见！")
            break
        
        if user_input.lower() in ("quit", "exit", "q"):
            print("👋 再见！")
            break
        
        reply = agent.chat(user_input)
        print(f"\n🤖: {reply}\n")
```

## ⚠️ 常见错误

### 错误 1：`ModuleNotFoundError: No module named 'tools'`
**原因**：Python 找不到 `src/tools/` 模块。
**解决**：
```bash
# 确保在 src/ 的父目录运行，或设置 PYTHONPATH
cd /path/to/project
PYTHONPATH=src python -m src.agent
```

### 错误 2：LLM 无限循环调用工具
**原因**：LLM 陷入循环，反复调用同一个工具。
**解决**：代码中已设置 `MAX_TOOL_CALLS = 10` 作为安全上限。你也可以在 system prompt 中提醒 LLM："不要重复调用同一个工具"。

### 错误 3：工具返回结果太长，API 报错
**原因**：某些命令（如 `cat` 大文件）输出过长，超出上下文窗口。
**解决**：`BashTool` 已设置 `MAX_OUTPUT_LENGTH = 10000` 自动截断。如果还不够，可以减小这个值。

### 错误 4：`json.decoder.JSONDecodeError`
**原因**：LLM 返回的工具调用参数不是合法 JSON。
**解决**：这通常是 LLM 模型的问题。尝试换用更强的模型（如 `gpt-4o`），或在 system prompt 中强调参数格式。

### 练习 1：添加 PythonTool
创建一个专门执行 Python 代码的工具（不是用 bash 调 `python3 -c`）。

**参考答案**：
```python
# src/tools/python_tool.py
from __future__ import annotations
import subprocess
from typing import Any
from tools.base import Tool

class PythonTool(Tool):
    name = "python"
    description = "执行 Python 代码片段。适用于测试代码、数据处理等。"
    
    @property
    def parameters(self) -> dict:
        return {
            "type": "object",
            "properties": {
                "code": {
                    "type": "string",
                    "description": "要执行的 Python 代码"
                }
            },
            "required": ["code"]
        }
    
    def execute(self, code: str, timeout: int = 10, **kwargs: Any) -> str:
        try:
            result = subprocess.run(
                ["python3", "-c", code],
                capture_output=True,
                text=True,
                timeout=timeout,
            )
            output = result.stdout
            if result.stderr:
                output += f"\n[stderr]\n{result.stderr}"
            if result.returncode != 0:
                output += f"\n[退出码: {result.returncode}]"
            return output[:5000]  # 限制输出
        except subprocess.TimeoutExpired:
            return f"[错误] 代码执行超时（{timeout}秒）"
        except Exception as e:
            return f"[错误] {e}"
```
