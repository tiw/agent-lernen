## 🧠 核心概念

### 为什么需要记忆管理？

前 4 章的 Agent 有一个致命问题：**消息历史会无限增长**。

```
对话 1: 用户问 → Agent 答（消息历史: 2 条）
对话 2: 用户问 → Agent 答（消息历史: 4 条）
对话 3: 用户问 → Agent 答（消息历史: 6 条）
...
对话 100: 消息历史 200 条 → Token 超限 → API 报错 💥
```

每个 LLM 都有**上下文窗口限制**：

| 模型 | 上下文窗口 | 大约相当于 |
|------|-----------|-----------|
| gpt-4o-mini | 128K tokens | ~100 页英文书 |
| gpt-4o | 128K tokens | ~100 页英文书 |
| Claude Sonnet | 200K tokens | ~150 页英文书 |
| Claude Opus | 200K tokens | ~150 页英文书 |

### Token 是什么？

Token 是 LLM 处理文本的基本单位。粗略估算：
- **英文**：1 token ≈ 4 个字符 ≈ 0.75 个单词
- **中文**：1 token ≈ 1.5 个汉字

```python
def estimate_tokens(text: str) -> int:
    """粗略估算 token 数量"""
    # 英文字符按 4 字符/token，中文按 1.5 字符/token
    english_chars = sum(1 for c in text if ord(c) < 128)
    chinese_chars = sum(1 for c in text if ord(c) >= 128)
    return english_chars // 4 + chinese_chars // 2
```

### 三层记忆管理架构

```
┌─────────────────────────────────────────────────┐
│  第 1 层：Token 估算与预算                        │
│  实时估算消息的 token 数量                        │
│  在发送前检查是否超限                             │
├─────────────────────────────────────────────────┤
│  第 2 层：上下文窗口管理                          │
│  消息滑动窗口（保留最近的 N 条）                   │
│  优先级保留（system prompt 永远保留）              │
├─────────────────────────────────────────────────┤
│  第 3 层：压缩策略                               │
│  当窗口快满时，压缩旧消息                         │
│  提取摘要、丢弃工具结果、保留关键信息              │
└─────────────────────────────────────────────────┘
```

### 代码 2：`src/memory/token_counter.py`

```python
"""
Token 估算器 —— 估算文本和消息的 token 数量。

注意：这是粗略估算。精确估算需要使用 tiktoken 库。
从零手写 AI Agent 课程 · 第 5 章
"""

from __future__ import annotations


def estimate_tokens(text: str) -> int:
    """
    粗略估算文本的 token 数量。
    
    估算规则：
    - 英文字符：4 字符 ≈ 1 token
    - 中文字符：1.5 字符 ≈ 1 token（约 2 字符/token）
    
    Args:
        text: 要估算的文本
        
    Returns:
        估算的 token 数量
    """
    if not text:
        return 0
    
    english_chars = 0
    chinese_chars = 0
    
    for char in text:
        if ord(char) < 128:
            english_chars += 1
        else:
            chinese_chars += 1
    
    # 英文 4 字符/token，中文 2 字符/token
    return english_chars // 4 + chinese_chars // 2


def count_message_tokens(messages: list[dict]) -> int:
    """
    估算消息列表的总 token 数量。
    
    每条消息除了内容 token 外，还有固定开销（角色名等）。
    OpenAI 的估算：每条消息额外 +4 tokens。
    
    Args:
        messages: 消息列表 [{"role": "...", "content": "..."}, ...]
        
    Returns:
        总 token 数量
    """
    total = 0
    
    for msg in messages:
        # 内容 token
        content = msg.get("content", "")
        total += estimate_tokens(content)
        
        # 角色名 token（system/user/assistant/tool 等）
        role = msg.get("role", "")
        total += estimate_tokens(role)
        
        # 每条消息的固定开销
        total += 4
        
        # tool_call_id 的额外开销
        if "tool_call_id" in msg:
            total += estimate_tokens(msg["tool_call_id"]) + 2
        
        # tool_calls 的额外开销
        if "tool_calls" in msg and msg["tool_calls"]:
            for tc in msg["tool_calls"]:
                total += estimate_tokens(str(tc)) + 5
    
    return total
```

### 代码 3：`src/memory/context_manager.py`

```python
"""
上下文窗口管理器 —— 管理消息历史，防止 Token 超限。

策略：
1. 预算检查：发送前检查 token 是否超限
2. 滑动窗口：保留最近的消息，丢弃最旧的
3. 压缩：提取摘要替代旧消息

从零手写 AI Agent 课程 · 第 5 章
"""

from __future__ import annotations

from typing import Optional

from memory.token_counter import count_message_tokens, estimate_tokens


class ContextManager:
    """
    上下文窗口管理器。
    
    核心职责：
    - 跟踪当前 token 使用量
    - 在超限时自动压缩消息历史
    - 保护关键消息（system prompt、最近对话）
    
    使用示例：
        manager = ContextManager(max_tokens=100_000)
        manager.add_message({"role": "user", "content": "你好"})
        if manager.needs_compression():
            manager.compress()
    """
    
    # 压缩时的目标使用率（压缩到窗口的 70%）
    COMPRESSION_TARGET = 0.7
    
    def __init__(
        self,
        max_tokens: int = 100_000,
        min_messages: int = 4,
        system_prompt_index: int = 0,
    ):
        """
        Args:
            max_tokens: 最大 token 数量（上下文窗口大小）
            min_messages: 最少保留的消息数（不包括 system prompt）
            system_prompt_index: system prompt 在消息列表中的索引
        """
        self.max_tokens = max_tokens
        self.min_messages = min_messages
        self.system_prompt_index = system_prompt_index
        self._compression_count = 0  # 压缩次数计数
    
    def count_tokens(self, messages: list[dict]) -> int:
        """计算消息列表的总 token 数"""
        return count_message_tokens(messages)
    
    def needs_compression(self, messages: list[dict]) -> bool:
        """
        检查是否需要压缩。
        
        Args:
            messages: 当前消息列表
            
        Returns:
            True 如果 token 数超过 max_tokens 的 90%
        """
        current_tokens = self.count_tokens(messages)
        return current_tokens > self.max_tokens * 0.9
    
    def compress(
        self,
        messages: list[dict],
        summary: Optional[str] = None,
    ) -> list[dict]:
        """
        压缩消息历史。
        
        压缩策略：
        1. 永远保留 system prompt
        2. 如果有摘要，用摘要替代旧消息
        3. 从最旧的消息开始丢弃，直到 token 数低于目标
        4. 至少保留 min_messages 条最近消息
        
        Args:
            messages: 当前消息列表
            summary: 可选的摘要文本（替代旧消息）
            
        Returns:
            压缩后的消息列表
        """
        self._compression_count += 1
        
        # 1. 分离 system prompt 和其他消息
        system_msg = messages[self.system_prompt_index]
        other_messages = [
            msg for i, msg in enumerate(messages)
            if i != self.system_prompt_index
        ]
        
        # 2. 构建压缩后的消息列表
        compressed = [system_msg]
        
        # 如果有摘要，加入摘要
        if summary:
            compressed.append({
                "role": "system",
                "content": f"[历史对话摘要]\n{summary}"
            })
        
        # 3. 从最旧的消息开始，逐条添加直到接近上限
        target_tokens = int(self.max_tokens * self.COMPRESSION_TARGET)
        current_tokens = self.count_tokens(compressed)
        
        # 至少保留最近的 min_messages 条
        messages_to_keep = other_messages[-self.min_messages:]
        messages_to_discard = other_messages[:-self.min_messages] if len(other_messages) > self.min_messages else []
        
        # 尝试添加可以容纳的旧消息
        for msg in messages_to_discard:
            msg_tokens = self.count_tokens([msg])
            if current_tokens + msg_tokens <= target_tokens:
                compressed.append(msg)
                current_tokens += msg_tokens
        
        # 添加必须保留的最近消息
        for msg in messages_to_keep:
            compressed.append(msg)
            current_tokens += self.count_tokens([msg])
        
        return compressed
    
    def get_stats(self, messages: list[dict]) -> dict:
        """
        获取上下文使用统计。
        
        Returns:
            包含 token 数、使用率、压缩次数等信息的字典
        """
        total_tokens = self.count_tokens(messages)
        
        # 按角色统计
        role_tokens = {}
        for msg in messages:
            role = msg.get("role", "unknown")
            content = msg.get("content", "")
            tokens = estimate_tokens(content) + 4  # +4 是角色开销
            role_tokens[role] = role_tokens.get(role, 0) + tokens
        
        return {
            "total_tokens": total_tokens,
            "max_tokens": self.max_tokens,
            "usage_percent": round(total_tokens / self.max_tokens * 100, 1),
            "message_count": len(messages),
            "compression_count": self._compression_count,
            "role_breakdown": role_tokens,
        }
```

```python
"""
AI Agent —— 支持工具调用 + 记忆管理
从零手写 AI Agent 课程 · 第 5 章

⚠️ 这是第 4 章 agent.py 的升级版。
   主要变更：
   - 新增 ContextManager 管理上下文窗口
   - chat() 方法加入压缩检查
   - 新增 get_context_stats() 方法
"""

from __future__ import annotations

import json
import os
from typing import Optional

from openai import OpenAI
from tools.base import Tool
from memory.context_manager import ContextManager


class Agent:
    """支持工具调用和记忆管理的 AI Agent"""
    
    MAX_TOOL_CALLS = 10
    
    def __init__(
        self,
        system_prompt: str = "你是一个有用的 AI 助手。",
        model: str = "gpt-4o-mini",
        tools: Optional[list[Tool]] = None,
        api_key: Optional[str] = None,
        max_context_tokens: int = 100_000,
    ):
        self.client = OpenAI(api_key=api_key or os.environ.get("OPENAI_API_KEY"))
        self.model = model
        
        # 消息历史
        self.messages = [
            {"role": "system", "content": system_prompt}
        ]
        
        # 工具
        self.tools = tools or []
        self.tool_map = {tool.name: tool for tool in self.tools}
        
        # 🆕 上下文管理器
        self.context_manager = ContextManager(
            max_tokens=max_context_tokens,
            min_messages=4,  # 至少保留最近 4 条消息
        )
    
    def chat(self, user_input: str) -> str:
        """与 Agent 对话（支持工具调用 + 记忆管理）"""
        self.messages.append({"role": "user", "content": user_input})
        
        # 🆕 检查是否需要压缩
        if self.context_manager.needs_compression(self.messages):
            print("  ⚠️  上下文窗口接近上限，执行压缩...")
            self.messages = self.context_manager.compress(self.messages)
        
        tool_defs = [t.to_openai_format() for t in self.tools] if self.tools else None
        tool_call_count = 0
        
        while True:
            response = self.client.chat.completions.create(
                model=self.model,
                messages=self.messages,
                tools=tool_defs,
            )
            
            choice = response.choices[0]
            message = choice.message
            
            if message.tool_calls:
                tool_call_count += 1
                
                if tool_call_count > self.MAX_TOOL_CALLS:
                    self.messages.append({
                        "role": "system",
                        "content": (
                            f"[警告] 工具调用次数已达上限（{self.MAX_TOOL_CALLS}次），"
                            "请停止调用工具并给出最终回复。"
                        )
                    })
                    continue
                
                self.messages.append(message)
                
                for tool_call in message.tool_calls:
                    tool_name = tool_call.function.name
                    tool_args = json.loads(tool_call.function.arguments)
                    
                    print(f"  🔧 调用工具: {tool_name}({tool_args})")
                    
                    if tool_name in self.tool_map:
                        result = self.tool_map[tool_name].execute(**tool_args)
                    else:
                        result = f"[错误] 未知工具: {tool_name}"
                    
                    self.messages.append({
                        "role": "tool",
                        "tool_call_id": tool_call.id,
                        "content": result,
                    })
                
                continue
            
            reply = message.content or ""
            self.messages.append({"role": "assistant", "content": reply})
            return reply
    
    def get_context_stats(self) -> dict:
        """获取当前上下文使用统计"""
        return self.context_manager.get_stats(self.messages)
    
    def reset(self):
        """重置对话历史"""
        self.messages = [self.messages[0]]
        self.context_manager = ContextManager(
            max_tokens=self.context_manager.max_tokens,
        )
```

## ⚠️ 常见错误

### 错误 1：Token 估算不准确
**原因**：我们的估算方法很粗略，实际 token 数可能有 20-30% 的偏差。
**解决**：这是可接受的。估算的目的是"防止超限"，不是"精确计算"。如果需要精确值，使用 `tiktoken` 库：
```python
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4o")
tokens = len(enc.encode(text))
```

### 错误 2：压缩后 Agent "忘记"了重要信息
**原因**：简单丢弃旧消息会丢失上下文。
**解决**：使用摘要压缩（需要调用 LLM 生成摘要），或者增加 `max_tokens`。

### 错误 3：`min_messages` 设置过大导致无法压缩
**原因**：如果最少保留消息数太多，压缩后仍然超限。
**解决**：确保 `min_messages` 足够小，让压缩后的 token 数低于 `max_tokens * 0.7`。
