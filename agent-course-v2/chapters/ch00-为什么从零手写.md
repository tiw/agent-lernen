## 🤔 为什么不用框架？

市面上有很多 Agent 框架：LangChain、LlamaIndex、AutoGen……为什么我们要从零手写？

### 一个真实的故事

假设你要用 LangChain 构建一个能读写文件、执行命令、搜索网络的 Agent。

```python
# LangChain 方式
from langchain.agents import initialize_agent
from langchain.tools import Tool

tools = [
    Tool(name="read_file", func=read_file, description="读取文件"),
    Tool(name="write_file", func=write_file, description="写入文件"),
    # ... 你需要为每个工具写包装
]

agent = initialize_agent(tools, llm, agent_type="zero-shot-react-description")
```

看起来很简单，对吧？但当你遇到以下问题时：

1. **工具调用顺序不对**——框架的内部调度逻辑是黑盒，你不知道为什么 Agent 先调了搜索再调文件读取
2. **错误处理不工作**——工具抛了异常，框架吞掉了，你找不到原因
3. **Token 超了**——框架没有帮你管理上下文窗口，LLM 返回 "context length exceeded"
4. **想加个安全扫描**——框架没有这个扩展点，你得改框架源码

这时候你会发现：**框架帮你省了 10 行代码，但花了你 10 小时调试。**

### 手写的好处

| 维度 | 框架 | 手写 |
|---|---|---|
| 上手速度 | ⚡ 快（几行代码跑起来） | 🐢 慢（需要理解每个组件） |
| 控制力 | 🔒 低（框架决定你能做什么） | 🔓 高（你决定一切） |
| 调试难度 | 🌑 黑盒 | ☀️ 每行代码都是你的 |
| 可扩展性 | 📦 受限 | ♾️ 无限 |

### 本课程的目标

**不是**教你造一个比 LangChain 更好的框架。

**而是**让你真正理解 Agent 的每一个组件是如何工作的。学完之后：
- 你能评估任何 Agent 框架的优劣
- 你能在框架不工作时自己修
- 你能根据自己的需求定制 Agent

## 🧠 核心概念：Agent 是什么？

Agent 的本质公式，记住这个就够了：

```
Agent = LLM + 循环 + 工具 + 记忆
```

| 组件 | 比喻 | 作用 |
|---|---|---|
| **LLM** | 大脑 | 思考和决策 |
| **循环** | 心脏 | 持续运行，直到任务完成 |
| **工具** | 手和脚 | 执行实际操作 |
| **记忆** | 经验 | 跨会话保持上下文 |

没有工具 → Agent 只能"说话"，不能"做事"
没有记忆 → Agent 每次都是"失忆"状态
没有循环 → Agent 只能回答一次，不能持续工作

## 💻 环境搭建

### 1. Python 环境

```bash
# 确认 Python 版本（需要 3.9+）
python3 --version
# 输出应该是 Python 3.9.x 或更高

# 创建项目目录
mkdir my-agent && cd my-agent

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate  # macOS/Linux
# 或 .venv\Scripts\activate  # Windows
```

### 2. 安装依赖

```bash
# 先装最小依赖（Ch01-04 只需要这些）
pip install openai anthropic

# 后续章节会逐步添加更多依赖
```

### 3. 配置 API 密钥

```bash
# OpenAI
export OPENAI_API_KEY="sk-..."

# 或 Anthropic
export ANTHROPIC_API_KEY="sk-ant-..."
```

> 💡 **没有 API 密钥？** 可以用免费的替代品：
> - [Ollama](https://ollama.com/) 本地运行开源模型
> - [Groq](https://groq.com/) 免费额度
> - 课程代码兼容任何 OpenAI 兼容的 API

### 4. 项目结构

```
my-agent/

├── skills/           # 技能系统（Ch08 开始）
```

## ⚠️ 常见错误

### 错误 1：Python 版本太低
```
SyntaxError: invalid syntax  # 在 dict[str, Any] 处报错
```
**原因**：你的 Python 版本低于 3.9。
**解决**：升级到 Python 3.9+，或在文件顶部加 `from __future__ import annotations`

### 错误 2：API 密钥未设置
```
AuthenticationError: No API key provided
```
**原因**：没有设置环境变量。
**解决**：`export OPENAI_API_KEY="sk-..."`，然后重启终端。

### 错误 3：虚拟环境未激活
```
ModuleNotFoundError: No module named 'openai'
```
**原因**：没有激活虚拟环境就运行代码。
**解决**：先 `source .venv/bin/activate`，再 `pip install openai`。
